文件编辑是 AI Agent 的核心能力，但也最容易造成意外破坏。本篇分析 Claude Code 的文件操作安全设计。

## 文件操作的风险

### 意外覆盖

```
用户：修改配置文件
AI：直接写入新内容
结果：原有配置丢失，系统崩溃
```

### 版本冲突

```
用户读取文件 (t0)
AI 编辑文件
外部修改文件 (t1 > t0)
AI 写入
结果：外部修改丢失
```

### 路径错误

```
用户：修改 .env
AI：写入 .env.local
结果：修改了错误文件
```

## Read-First 规则

### 核心原则

**必须先读取，才能编辑**

```typescript
async function editFile(input: EditInput, context: ToolContext): ToolResult {
  // 1. 检查是否已读取
  const readState = context.readFileState.get(input.file_path)

  if (!readState) {
    return {
      success: false,
      error: 'File has not been read yet. Read it first before editing.',
      errorCode: 6,
    }
  }

  // 2. 检查是否是完整读取
  if (readState.isPartialView) {
    return {
      success: false,
      error: 'File was partially read. Read the full file before editing.',
      errorCode: 6,
    }
  }

  // 3. 继续编辑...
}
```

### 为什么必须先读？

1. **了解文件内容**：AI 知道当前状态
2. **检测外部修改**：对比时间戳
3. **防止盲目覆盖**：有意识地编辑
4. **记录读取状态**：用于版本控制

## 文件状态缓存

### 状态记录

```typescript
type FileState = {
  content: string        // 文件内容
  timestamp: number      // 读取时间 (mtime)
  offset?: number        // 读取偏移（部分读取）
  isPartialView: boolean // 是否部分读取
}

const readFileState: Map<string, FileState> = new Map()
```

### 读取时记录

```typescript
async function readFile(path: string, context: ToolContext): ToolResult {
  // 读取文件
  const content = await fs.readFile(path, 'utf-8')

  // 获取修改时间
  const stats = await fs.stat(path)
  const timestamp = stats.mtimeMs

  // 记录状态
  context.readFileState.set(path, {
    content,
    timestamp,
    isPartialView: false,
  })

  return { success: true, content }
}
```

### 部分读取标记

```typescript
async function readFileRange(path: string, offset: number, limit: number): ToolResult {
  const content = await fs.readFile(path, 'utf-8')
  const partialContent = content.slice(offset, offset + limit)

  // 部分读取标记
  context.readFileState.set(path, {
    content: partialContent,
    timestamp: getMtime(path),
    offset,
    isPartialView: true,  // 标记为部分读取
  })

  return { success: true, content: partialContent }
}
```

**效果**：部分读取的文件不能编辑。

## 外部修改检测

### 时间戳检查

```typescript
async function checkExternalModification(path: string, readState: FileState): boolean {
  const currentMtime = await getMtime(path)

  // 如果修改时间变化
  if (currentMtime > readState.timestamp) {
    // 进一步检查内容是否真的变了
    const currentContent = await fs.readFile(path, 'utf-8')

    if (currentContent !== readState.content) {
      return true  // 内容确实变了
    }

    // 内容没变（可能是云同步、杀毒软件等触发 mtime 变化）
    return false
  }

  return false  // 未修改
}
```

### 处理外部修改

```typescript
async function editFile(input: EditInput, context: ToolContext): ToolResult {
  const readState = context.readFileState.get(input.file_path)

  // 检查外部修改
  if (await checkExternalModification(input.file_path, readState)) {
    return {
      success: false,
      error: 'File has been modified since you read it. Read it again to see current content.',
      errorCode: 7,
    }
  }

  // 安全编辑
  await applyEdit(input)
}
```

## Notebook 重定向

### 特殊文件处理

`.ipynb` 文件不能用 Edit 工具：

```typescript
if (input.file_path.endsWith('.ipynb')) {
  return {
    success: false,
    error: 'Jupyter Notebooks must use NotebookEdit tool, not Edit.',
    errorCode: 5,
  }
}
```

**原因**：Notebook 有特殊的 JSON 结构，普通编辑可能破坏。

### NotebookEdit 工具

专门处理 `.ipynb`：

```typescript
async function notebookEdit(input: NotebookEditInput): ToolResult {
  // 读取 Notebook
  const notebook = await readNotebook(input.notebook_path)

  // 编辑特定 cell
  if (input.edit_mode === 'replace') {
    notebook.cells[input.cell_number] = {
      cell_type: input.cell_type,
      source: input.new_source,
    }
  }

  // 写回 Notebook
  await writeNotebook(input.notebook_path, notebook)
}
```

## 路径验证

### 路径规范化

```typescript
function normalizePath(path: string): string {
  // 解析相对路径
  const resolved = path.startsWith('/') ? path : resolve(cwd, path)

  // 规范化（移除 ../ 等）
  const normalized = normalize(resolved)

  return normalized
}
```

### 安全路径检查

```typescript
const BLOCKED_PATHS = new Set([
  '/dev/zero',
  '/dev/random',
  '/dev/urandom',
  '/dev/tty',
  '/dev/console',
])

function isBlockedPath(path: string): boolean {
  return BLOCKED_PATHS.has(normalizePath(path))
}
```

### 权限边界

```typescript
function checkPathPermission(path: string, allowedPaths: string[]): boolean {
  const normalized = normalizePath(path)

  // 检查是否在允许范围内
  return allowedPaths.some(allowed => {
    return normalized.startsWith(allowed)
  })
}
```

## Write 工具的安全

Write 工具（创建新文件）也需要安全检查：

### 不允许覆盖已读文件

```typescript
async function writeFile(input: WriteInput, context: ToolContext): ToolResult {
  const path = normalizePath(input.file_path)

  // 如果文件已存在且已读取，必须用 Edit
  if (fs.existsSync(path) && context.readFileState.has(path)) {
    return {
      success: false,
      error: 'File exists and was read. Use Edit tool to modify existing files.',
      errorCode: 8,
    }
  }

  // 创建新文件
  await fs.writeFile(path, input.content)
}
```

### 新文件确认

```typescript
// 创建新文件前确认
if (!fs.existsSync(path)) {
  // 显示确认对话框
  const confirmed = await confirmCreate(path)

  if (!confirmed) {
    return { success: false, error: 'User denied file creation' }
  }
}
```

## 文件类型检测

### 二进制文件阻止

```typescript
async function detectFileType(path: string): FileType {
  const buffer = await fs.readFile(path)
  const magic = buffer.slice(0, 4)

  // 检查 magic number
  if (magic.equals(Buffer.from([0x89, 0x50, 0x4E, 0x47]))) {
    return 'image/png'
  }

  if (magic.equals(Buffer.from([0xFF, 0xD8, 0xFF]))) {
    return 'image/jpeg'
  }

  // 检查是否是二进制
  const isBinary = hasBinaryContent(buffer)

  if (isBinary) {
    return 'binary'
  }

  return 'text'
}

// 二进制文件不能 Edit
if (fileType === 'binary' && !fileType.startsWith('image/')) {
  return {
    success: false,
    error: 'Cannot edit binary files',
    errorCode: 9,
  }
}
```

**例外**：图片文件可以读取（返回 base64），但不能编辑。

## 安全编辑流程

### 完整流程图

```
Edit 请求
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 路径规范化                                                │
│    - 解析相对路径                                            │
│    - 移除 ../                                                │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 检查是否已读取                                            │
│    - readFileState.has(path)?                               │
│    - 否 → 错误：必须先读取                                    │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 检查是否完整读取                                          │
│    - isPartialView?                                         │
│    - 是 → 错误：需要完整读取                                  │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 检查文件类型                                              │
│    - .ipynb → 重定向到 NotebookEdit                          │
│    - 二进制 → 错误                                            │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 检查外部修改                                              │
│    - mtime > readTimestamp?                                 │
│    - 内容变化?                                               │
│    - 是 → 错误：需要重新读取                                  │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 执行编辑                                                  │
│    - 应用 old_string → new_string                            │
│    - 写入文件                                                │
│    - 更新 readFileState                                      │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
成功返回
```

## 错误码参考

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 5 | Notebook 文件 | 使用 NotebookEdit |
| 6 | 未读取文件 | 先 Read 再 Edit |
| 7 | 外部修改 | 重新 Read |
| 8 | 已存在文件 | 使用 Edit |
| 9 | 二进制文件 | 无法编辑 |
