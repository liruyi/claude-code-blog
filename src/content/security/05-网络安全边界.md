AI Agent 可以访问网络，但需要安全边界。本篇分析 WebFetch 和 WebSearch 的限制策略。

## 网络访问风险

### 数据泄露风险

```
场景：用户有敏感代码
AI: WebFetch 到恶意网站 → 敏感信息被发送
结果：数据泄露
```

### SSRF 风险

```
场景：企业内网
AI: WebFetch http://internal-admin.local → 访问内网资源
结果：绕过防火墙，访问敏感内网
```

### 恶意内容风险

```
场景：恶意网站返回 prompt injection
AI: WebFetch 恶意网站 → 网站内容注入恶意指令
结果：AI 被操控执行危险操作
```

## URL 验证

### 黑名单机制

```typescript
const BLOCKED_DOMAINS = new Set([
  // 内网地址（SSRF 防护）
  'localhost',
  '127.0.0.1',
  '0.0.0.0',
  '::1',
  'internal.',
  'admin.',
  'private.',

  // 敏感服务
  'aws.amazon.com',
  'console.aws.amazon.com',
  'management.azure.com',
  'console.cloud.google.com',

  // 认证服务
  'accounts.google.com',
  'login.microsoftonline.com',
  'auth0.com',
])

function isBlockedUrl(url: string): boolean {
  try {
    const parsed = new URL(url)
    const hostname = parsed.hostname.toLowerCase()

    // 直接匹配
    if (BLOCKED_DOMAINS.has(hostname)) {
      return true
    }

    // 子域名匹配
    for (const blocked of BLOCKED_DOMAINS) {
      if (hostname.endsWith('.' + blocked)) {
        return true
      }
    }

    // IP 地址检查
    if (isPrivateIP(hostname)) {
      return true
    }

    return false
  } catch {
    return true  // 无效 URL，拒绝
  }
}
```

### 内网 IP 检测

```typescript
function isPrivateIP(hostname: string): boolean {
  // 检查是否是 IP 地址
  const ipPattern = /^(\d{1,3}\.){3}\d{1,3}$/

  if (!ipPattern.test(hostname)) {
    return false
  }

  const parts = hostname.split('.').map(Number)

  // 10.0.0.0/8
  if (parts[0] === 10) return true

  // 172.16.0.0/12
  if (parts[0] === 172 && parts[1] >= 16 && parts[1] <= 31) return true

  // 192.168.0.0/16
  if (parts[0] === 192 && parts[1] === 168) return true

  // 169.254.0.0/16 (link-local)
  if (parts[0] === 169 && parts[1] === 254) return true

  return false
}
```

### URL 重定向检测

```typescript
async function checkRedirectSafety(url: string): SafeUrlResult {
  try {
    const response = await fetch(url, { method: 'HEAD', redirect: 'manual' })

    // 检查重定向
    if (response.status === 301 || response.status === 302 || response.status === 303) {
      const redirectUrl = response.headers.get('location')

      if (redirectUrl) {
        // 检查重定向目标是否安全
        if (isBlockedUrl(redirectUrl)) {
          return {
            safe: false,
            reason: `重定向到黑名单地址: ${redirectUrl}`,
          }
        }

        // 递归检查（最多 5 次）
        return checkRedirectSafety(redirectUrl)
      }
    }

    return { safe: true }
  } catch (error) {
    return { safe: false, reason: 'URL 检查失败' }
  }
}
```

## WebFetch 安全流程

### 完整检查流程

```typescript
async function webFetchTool(input: WebFetchInput): ToolResult {
  const url = input.url

  // 1. URL 格式验证
  if (!isValidUrl(url)) {
    return {
      success: false,
      error: 'URL 格式无效',
    }
  }

  // 2. 黑名单检查
  if (isBlockedUrl(url)) {
    return {
      success: false,
      error: 'URL 在黑名单中，禁止访问',
      reason: getBlockedReason(url),
    }
  }

  // 3. 重定向检查
  const redirectCheck = await checkRedirectSafety(url)
  if (!redirectCheck.safe) {
    return {
      success: false,
      error: redirectCheck.reason,
    }
  }

  // 4. 权限检查
  const permission = await checkPermission('WebFetch', { url })
  if (permission.behavior === 'deny') {
    return {
      success: false,
      error: '权限拒绝',
    }
  }

  // 5. 执行请求
  try {
    const content = await safeFetch(url)

    // 6. 内容检查
    const contentCheck = checkContentSafety(content)
    if (!contentCheck.safe) {
      return {
        success: false,
        error: '内容安全检查失败',
        reason: contentCheck.reason,
      }
    }

    // 7. 处理内容
    const processed = processWebContent(content, input.prompt)

    return {
      success: true,
      content: processed,
    }
  } catch (error) {
    return {
      success: false,
      error: `请求失败: ${error.message}`,
    }
  }
}
```

### 安全请求

```typescript
async function safeFetch(url: string): string {
  const response = await fetch(url, {
    method: 'GET',
    headers: {
      'User-Agent': 'Claude-Code-CLI/2.1',
      'Accept': 'text/html,application/xhtml+xml,text/plain',
    },
    timeout: 30000,  // 30 秒超时
    redirect: 'follow',  // 最多跟随 5 次重定向
    sizeLimit: 1000000,  // 1MB 限制
  })

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`)
  }

  const content = await response.text()

  // 大内容截断
  if (content.length > 500000) {
    return content.slice(0, 500000) + '\n... (内容已截断)'
  }

  return content
}
```

## WebSearch 安全

### 搜索限制

```typescript
const SEARCH_ALLOWED_DOMAINS = [
  // 可信技术文档
  'docs.python.org',
  'developer.mozilla.org',
  'react.dev',
  'nodejs.org',
  'npmjs.com',
  'github.com',
  'stackoverflow.com',
  'medium.com',
  'dev.to',

  // 新闻（有限制）
  'news.ycombinator.com',
]

const SEARCH_BLOCKED_DOMAINS = [
  // 恶意网站
  'malware',
  'phishing',
  'spam',
]

function checkSearchQuery(query: string): SafeQueryResult {
  // 检查查询中的敏感词
  const sensitivePatterns = [
    /password/i,
    /secret/i,
    /token/i,
    /api[_-]?key/i,
    /credential/i,
  ]

  for (const pattern of sensitivePatterns) {
    if (pattern.test(query)) {
      return {
        safe: false,
        reason: '搜索查询包含敏感信息',
      }
    }
  }

  return { safe: true }
}
```

### 搜索结果过滤

```typescript
function filterSearchResults(results: SearchResult[]): SearchResult[] {
  return results.filter(result => {
    const url = result.url

    // 黑名单检查
    if (SEARCH_BLOCKED_DOMAINS.some(blocked => url.includes(blocked))) {
      return false
    }

    // 内容检查
    const contentCheck = checkContentSafety(result.snippet)
    if (!contentCheck.safe) {
      return false
    }

    return true
  })
}
```

## 内容安全检查

### Prompt Injection 检测

```typescript
const PROMPT_INJECTION_PATTERNS = [
  /ignore previous instructions/i,
  /system prompt/i,
  /you are now/i,
  /forget everything/i,
  /disregard all/i,
  /new directive/i,
  /override/i,
  /admin mode/i,
  /debug mode/i,
]

function checkPromptInjection(content: string): SafeContentResult {
  for (const pattern of PROMPT_INJECTION_PATTERNS) {
    if (pattern.test(content)) {
      return {
        safe: false,
        reason: '检测到可能的 prompt injection',
        pattern: pattern.toString(),
      }
    }
  }

  return { safe: true }
}
```

### 恶意内容检测

```typescript
function checkContentSafety(content: string): SafeContentResult {
  // 1. Prompt injection 检测
  const injectionCheck = checkPromptInjection(content)
  if (!injectionCheck.safe) {
    return injectionCheck
  }

  // 2. 恶意脚本检测
  const scriptCheck = checkMaliciousScript(content)
  if (!scriptCheck.safe) {
    return scriptCheck
  }

  // 3. 异常内容检测
  const anomalyCheck = checkContentAnomaly(content)
  if (!anomalyCheck.safe) {
    return anomalyCheck
  }

  return { safe: true }
}

function checkMaliciousScript(content: string): SafeContentResult {
  // 检查危险脚本模式
  const dangerousPatterns = [
    /<script[^>]*>[\s\S]*<\/script>/gi,
    /javascript:/gi,
    /on\w+\s*=/gi,  // onclick, onload 等
    /data:/gi,
  ]

  for (const pattern of dangerousPatterns) {
    if (pattern.test(content)) {
      return {
        safe: false,
        reason: '检测到可能的恶意脚本',
      }
    }
  }

  return { safe: true }
}
```

### 内容异常检测

```typescript
function checkContentAnomaly(content: string): SafeContentResult {
  // 检查异常长度
  if (content.length > 5000000) {  // 5MB
    return {
      safe: false,
      reason: '内容过大，可能有问题',
    }
  }

  // 检查异常字符
  const highEntropyPattern = /[^\x00-\x7F]{50,}/
  if (highEntropyPattern.test(content)) {
    return {
      safe: false,
      reason: '检测到异常高熵内容，可能是加密数据',
    }
  }

  // 检查重复内容
  if (isHighlyRepetitive(content)) {
    return {
      safe: false,
      reason: '内容高度重复，可能是攻击',
    }
  }

  return { safe: true }
}
```

## 错误处理

### 安全错误分类

| 错误类型 | 说明 | 处理 |
|----------|------|------|
| URL 黑名单 | 访问禁止地址 | 直接拒绝，不提示 |
| SSRF 检测 | 内网地址 | 拒绝，警告用户 |
| Prompt Injection | 检测到注入 | 拒绝，记录日志 |
| 大内容 | 超过限制 | 截断处理 |
| 请求失败 | 网络/服务器错误 | 提示用户，可重试 |

### 用户提示

```
❌ URL 黑名单
无法访问该 URL：安全策略禁止访问内网地址

❌ SSRF 检测
警告：该 URL 可能指向内部服务，禁止访问

❌ Prompt Injection
安全检查失败：该页面可能包含恶意内容，已拒绝处理

❌ 大内容
内容过大，已截断至前 500KB
```
