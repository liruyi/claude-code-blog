上一篇介绍了威胁模型。本篇深入 Claude Code 的工具权限系统，看看分层防御如何实现。

## 权限检查的复杂性

简单权限：Yes/No

```
用户: 读取文件
系统: 允许 ✓
```

复杂权限：Why/How/When

```
用户: 执行 rm -rf node_modules
系统:
  - 这个命令安全吗？
  - 用户之前批准过类似的吗？
  - 企业策略允许吗？
  - AI 分类器怎么判断？
  - 显示对话框确认？
```

## 四层权限架构

Claude Code 的权限检查分四层：

```
工具调用请求
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ 第一层：工具自身检查                                       │
│ - inputSchema 验证                                       │
│ - 内置安全规则                                            │
│ 例如：Read 不允许读取 /dev/zero                          │
└─────────────────────────────────────────────────────────┘
    │ 通过
    ▼
┌─────────────────────────────────────────────────────────┐
│ 第二层：规则匹配                                          │
│ - 用户自定义规则                                          │
│ - 企业策略规则                                            │
│ 例如：用户设置 Bash(git *) 自动允许                      │
└─────────────────────────────────────────────────────────┘
    │ 未匹配
    ▼
┌─────────────────────────────────────────────────────────┐
│ 第三层：权限模式检查                                       │
│ - auto：自动允许                                          │
│ - acceptEdits：只允许编辑                                 │
│ - plan：不允许执行                                        │
│ 例如：plan 模式下只允许 Read/Grep                        │
└─────────────────────────────────────────────────────────┘
    │ 未自动允许
    ▼
┌─────────────────────────────────────────────────────────┐
│ 第四层：AI 分类器                                         │
│ - 高置信度允许 → 自动允许                                 │
│ - 高置信度拒绝 → 显示警告                                 │
│ - 不确定 → 显示对话框                                     │
└─────────────────────────────────────────────────────────┘
    │ 不确定
    ▼
┌─────────────────────────────────────────────────────────┐
│ 第五层：权限对话框                                        │
│ - 用户显式批准                                            │
│ - 显示操作详情                                            │
│ - 允许/拒绝/会话允许                                      │
└─────────────────────────────────────────────────────────┘
    │ 用户批准
    ▼
工具执行
```

**关键洞察**：每一层都可能自动允许，减少用户打扰。

## 第一层：工具自身检查

每个工具有自己的安全检查：

### FileReadTool 检查

```typescript
// 1. 设备文件阻止
const BLOCKED_DEVICE_PATHS = new Set([
  '/dev/zero', '/dev/random', '/dev/urandom',
  '/dev/tty', '/dev/console', '/dev/stdin',
])

// 2. 路径验证
if (BLOCKED_DEVICE_PATHS.has(normalizedPath)) {
  return { behavior: 'deny', message: 'Cannot read device files' }
}

// 3. 文件类型检查（使用 magic number）
const fileType = await detectFileType(filePath)
if (fileType === 'binary' && !isImage(fileType)) {
  return { behavior: 'deny', message: 'Cannot read binary files' }
}
```

### FileEditTool 检查

```typescript
// 1. 必须先读取
const readTimestamp = readFileState.get(filePath)
if (!readTimestamp) {
  return {
    behavior: 'deny',
    message: 'File has not been read yet. Read it first.',
  }
}

// 2. 文件未修改
if (lastWriteTime > readTimestamp.timestamp) {
  // 检查内容是否真的变了
  const currentContent = await readFile(filePath)
  if (currentContent !== readTimestamp.content) {
    return {
      behavior: 'deny',
      message: 'File has been modified since read',
    }
  }
}

// 3. Notebook 重定向
if (filePath.endsWith('.ipynb')) {
  return {
    behavior: 'deny',
    message: 'Use NotebookEdit tool for .ipynb files',
  }
}
```

### BashTool 检查（14 步）

Bash 权限检查是最复杂的，有 14 个步骤：

```
1. AST 安全解析（tree-sitter）
2. 复杂命令处理（命令替换、控制流）
3. 语义级检查（zsh builtins, eval）
4. 沙箱自动允许检查
5. Bash deny/ask classifier 并行检查
6. 命令操作符检查（管道、重定向）
7. 传统安全检查
8. 命令分割 + cd 过滤
9. 子命令数限制
10. 多 cd 命令检查
11. cd + git 组合检查（防止 bare repo RCE）
12. 子命令权限检查
13. 路径约束检查
14. 最终决策
```

## 第二层：规则匹配

用户和企业可以定义规则：

### 规则格式

```typescript
type PermissionRule = {
  toolName: string        // 工具名，支持通配符
  input?: object          // 输入匹配
  behavior: 'allow' | 'deny' | 'ask'
  reason?: string         // 原因说明
}
```

### 用户规则示例

```json
{
  'permissions': {
    'allow': [
      { 'toolName': 'Bash', 'input': { 'command': 'git *' }, 'behavior': 'allow' },
      { 'toolName': 'Read', 'behavior': 'allow' }
    ],
    'deny': [
      { 'toolName': 'Bash', 'input': { 'command': 'rm -rf *' }, 'behavior': 'deny' }
    ]
  }
}
```

### 企业规则示例

```json
{
  'policySettings': {
    'allowedTools': ['Read', 'Grep', 'Glob', 'WebFetch'],
    'deniedTools': ['Bash'],
    'allowedMcpServers': [{ 'serverName': 'slack' }]
  }
}
```

### 规则匹配逻辑

```typescript
function matchRule(rule: PermissionRule, toolName: string, input: object): boolean {
  // 1. 工具名匹配
  if (!matchToolName(rule.toolName, toolName)) return false

  // 2. 输入匹配（如果有）
  if (rule.input && !matchInput(rule.input, input)) return false

  return true
}

// 工具名匹配支持通配符
function matchToolName(pattern: string, name: string): boolean {
  // 'Bash' → 精确匹配
  // 'Bash(git *)' → 工具名 + 输入模式
  // '*' → 所有工具
  // 'Bash|Edit' → 多个工具

  if (pattern === '*') return true
  if (pattern.includes('|')) {
    return pattern.split('|').some(p => p === name)
  }
  return pattern === name
}
```

## 第三层：权限模式检查

Claude Code 有多种权限模式：

| 模式 | 说明 | 工具限制 |
|------|------|----------|
| `auto` | 自动允许所有 | 无限制 |
| `acceptEdits` | 自动接受编辑 | Edit/Write 自动允许 |
| `plan` | 规划模式 | 只允许 Read/Grep/Glob |

### 模式实现

```typescript
function checkPermissionMode(mode: PermissionMode, tool: Tool): PermissionResult {
  if (mode === 'auto') {
    return { behavior: 'allow' }
  }

  if (mode === 'acceptEdits') {
    if (tool.name === 'Edit' || tool.name === 'Write') {
      return { behavior: 'allow' }
    }
  }

  if (mode === 'plan') {
    if (tool.isReadOnly()) {
      return { behavior: 'allow' }
    }
    return { behavior: 'deny', message: 'Plan mode: only read operations allowed' }
  }

  return { behavior: 'ask' }  // 需要对话框
}
```

## 第四层：AI 分类器

Claude Code 使用 AI 分类器判断命令安全性：

### 分类器调用

```typescript
async function classifyBashCommand(
  command: string,
  cwd: string,
  allowDescriptions: boolean,
  mode: 'allow' | 'deny'
): Promise<ClassificationResult> {
  // 调用 Anthropic API 分类命令
  const response = await anthropic.messages.create({
    model: 'claude-3-haiku-20240307',
    max_tokens: 100,
    system: CLASSIFIER_SYSTEM_PROMPT,
    messages: [{
      role: 'user',
      content: `Command: ${command}\nDirectory: ${cwd}\nClassify: ${mode}`
    }]
  })

  // 解析结果
  return {
    classification: response.content[0].text,  // 'allow' | 'deny' | 'ask'
    confidence: parseFloat(response.content[1].text),
    reason: response.content[2].text,
  }
}
```

### 推测执行（Speculative Check）

用户看到对话框时，后台并行运行分类器：

```typescript
// 在权限对话框显示时启动
function startSpeculativeClassifierCheck(command: string, context: ToolUseContext) {
  const promise = classifyBashCommand(command, cwd, false, 'allow')
  speculativeChecks.set(command, promise)
}

// 如果分类器高置信度允许，自动消失对话框
async function checkSpeculativeResult(command: string): Promise<boolean> {
  const result = await speculativeChecks.get(command)
  if (result.confidence > 0.95 && result.classification === 'allow') {
    return true  // 自动允许
  }
  return false
}
```

**效果**：用户看到对话框，但可能几秒后自动消失（因为分类器允许）。

## 第五层：权限对话框

最后防线是用户确认：

### 对话框内容

```
┌────────────────────────────────────────────────────────────┐
│ ⚠️  Claude 想要执行 Bash 命令                               │
│                                                            │
│ 命令:                                                      │
│ rm -rf node_modules                                        │
│                                                            │
│ 影响:                                                      │
│ 这将删除 500MB 文件                                         │
│                                                            │
│ 推理:                                                      │
│ 清理项目依赖目录                                            │
│                                                            │
│ ┌────────────────────────────────────────────────────────┐ │
│ │ [允许一次] [允许本次会话] [拒绝] [查看详情]              │ │
│ └────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────┘
```

### 用户选择

| 选择 | 效果 |
|------|------|
| 允许一次 | 只允许这次执行 |
| 允许本次会话 | 同类命令自动允许 |
| 拒绝 | 阻止执行，显示原因 |
| 查看详情 | 显示完整工具输入 |

### 会话规则记忆

用户选择'允许本次会话'后，自动添加临时规则：

```typescript
sessionAllowRules.push({
  toolName: 'Bash',
  input: { command: similarPattern },
  behavior: 'allow',
  reason: 'User allowed in session',
})
```

## 权限结果传递

权限检查结果传递给工具：

```typescript
type PermissionResult = {
  behavior: 'allow' | 'deny' | 'ask'
  message?: string           // 显示给用户的原因
  updatedInput?: object      // 可能修改输入（如路径修正）
  decisionReason?: string    // 内部决策原因
}
```

### 结果处理

```typescript
async function executeTool(tool: Tool, input: object, context: ToolUseContext) {
  const permissionResult = await checkPermission(tool, input, context)

  if (permissionResult.behavior === 'deny') {
    return {
      type: 'tool_result',
      tool_use_id: generateId(),
      content: `Permission denied: ${permissionResult.message}`,
      is_error: true,
    }
  }

  if (permissionResult.behavior === 'ask') {
    const userDecision = await showPermissionDialog(tool, input, permissionResult)
    if (userDecision.denied) {
      return { ... }  // 同 deny
    }
  }

  // 允许执行
  return await tool.call(input, context)
}
```
