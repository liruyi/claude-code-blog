## 案例 1：GitLab PR 审批流程

**需求**：
- 自动检查 PR 代码质量
- 审批后发送通知
- 记录审批日志

### MCP Server：GitLab

```typescript
import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js'
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js'
import { z } from 'zod'

const GITLAB_URL = process.env.GITLAB_URL
const GITLAB_TOKEN = process.env.GITLAB_TOKEN

const server = new McpServer({
  name: 'gitlab-server',
  version: '1.0.0',
})

server.tool(
  'get_pr',
  'Get PR details',
  {
    project_id: z.string(),
    pr_id: z.number(),
  },
  async ({ project_id, pr_id }) => {
    const response = await fetch(
      `${GITLAB_URL}/api/v4/projects/${project_id}/merge_requests/${pr_id}`,
      {
        headers: { 'PRIVATE-TOKEN': GITLAB_TOKEN },
      }
    )
    const data = await response.json()
    return {
      content: [{ type: 'text', text: JSON.stringify(data) }],
    }
  }
)

server.tool(
  'approve_pr',
  'Approve a PR',
  {
    project_id: z.string(),
    pr_id: z.number(),
  },
  async ({ project_id, pr_id }) => {
    const response = await fetch(
      `${GITLAB_URL}/api/v4/projects/${project_id}/merge_requests/${pr_id}/approve`,
      {
        method: 'POST',
        headers: { 'PRIVATE-TOKEN': GITLAB_TOKEN },
      }
    )
    const data = await response.json()
    return {
      content: [{ type: 'text', text: JSON.stringify(data) }],
    }
  }
)

async function main() {
  const transport = new StdioServerTransport()
  await server.connect(transport)
}

main()
```

### Skill：审批流程

```markdown
---
name: review-pr
description: Review and approve a GitLab PR
allowedTools: [mcp__gitlab__*, Read, Grep]
argumentHint: [project_id/pr_id]
---

# Review PR Skill

Review and approve a GitLab PR following team standards.

## Steps

1. Parse project_id and pr_id from argument
2. Get PR details: `mcp__gitlab__get_pr`
3. Analyze changed files:
   - Read key files
   - Check code quality
   - Identify security issues
4. If passes review:
   - Approve: `mcp__gitlab__approve_pr`
   - Report approval
5. If issues found:
   - Report issues with suggestions
   - Do not approve

## Review Criteria

- Code follows team style guide
- No obvious security vulnerabilities
- Tests are included for new features
- Documentation updated if needed
```

### Hook：审批后通知

```bash
#!/bin/bash
# pr-approved-notify.sh

input=$(cat)
tool_name=$(echo '$input' | jq -r '.tool_name')

if [[ '$tool_name' == 'mcp__gitlab__approve_pr' ]]; then
  project=$(echo '$input' | jq -r '.tool_input.project_id')
  pr=$(echo '$input' | jq -r '.tool_input.pr_id')

  # Slack 通知
  curl -s -X POST '$SLACK_WEBHOOK_URL' \
    -H 'Content-type: application/json' \
    -d '{\'text\': \'✅ PR #$pr approved in $project by Claude Code\'}'

  # 审计日志
  echo '$(date -Iseconds) | APPROVED | $project/$pr' >> ~/.claude/approval.log
fi

echo '{'continue': true}'
```

## 案例 2：Jira 任务自动创建

**需求**：
- 根据用户描述创建 Jira 任务
- 自动分配负责人
- 添加到项目

### MCP Server：Jira

```typescript
server.tool(
  'create_issue',
  'Create a Jira issue',
  {
    project: z.string().describe('Project key'),
    summary: z.string().describe('Issue summary'),
    description: z.string().describe('Issue description'),
    type: z.enum(['Bug', 'Task', 'Story']).describe('Issue type'),
    assignee: z.string().optional().describe('Assignee username'),
  },
  async ({ project, summary, description, type, assignee }) => {
    const response = await fetch(
      `${JIRA_URL}/rest/api/3/issue`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${JIRA_TOKEN}`,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          fields: {
            project: { key: project },
            summary,
            description,
            issuetype: { name: type },
            assignee: assignee ? { name: assignee } : undefined,
          },
        }),
      }
    )
    const data = await response.json()
    return {
      content: [{ type: 'text', text: `Created issue ${data.key}: ${data.self}` }],
    }
  }
)
```

## 案例 3：知识库集成

**需求**：
- 企业内部文档知识库
- 搜索和查询
- 添加新文档

### MCP Server：知识库

```typescript
const KNOWLEDGE_DIR = '/data/knowledge'

server.tool(
  'search',
  'Search knowledge base',
  {
    query: z.string(),
    limit: z.number().optional().default(5),
  },
  async ({ query, limit }) => {
    const files = await fs.readdir(KNOWLEDGE_DIR)
    const results = []

    for (const file of files) {
      const content = await fs.readFile(path.join(KNOWLEDGE_DIR, file))
      const matches = content.toLowerCase().includes(query.toLowerCase())

      if (matches) {
        results.push({
          file,
          snippet: content.slice(0, 300),
        })
      }
      if (results.length >= limit) break
    }

    return {
      content: [{ type: 'text', text: JSON.stringify(results) }],
    }
  }
)

server.tool(
  'add_doc',
  'Add document to knowledge base',
  {
    filename: z.string(),
    content: z.string(),
  },
  async ({ filename, content }) => {
    await fs.writeFile(path.join(KNOWLEDGE_DIR, filename), content)
    return {
      content: [{ type: 'text', text: `Added ${filename}` }],
    }
  }
)

// Resources: 知识库目录
server.resources(
  async () => ({
    resources: [
      {
        uri: 'kb://index',
        name: 'Knowledge Base Index',
        mimeType: 'application/json',
      },
    ],
  })
)

server.readResource(
  async (uri) => {
    if (uri === 'kb://index') {
      const files = await fs.readdir(KNOWLEDGE_DIR)
      return {
        contents: [{
          uri,
          mimeType: 'application/json',
          text: JSON.stringify(files),
        }],
      }
    }
    throw new Error(`Unknown resource: ${uri}`)
  }
)
```

## 案例 4：CI/CD 触发 Hook

**需求**：
- 特定操作后触发 CI/CD
- 记录触发日志

### Hook：提交后触发

```bash
#!/bin/bash
# trigger-ci.sh

input=$(cat)
command=$(echo '$input' | jq -r '.tool_input.command')

# 检查 git commit
if [[ '$command' =~ 'git commit' ]]; then
  branch=$(git branch --show-current)

  # 触发 CI
  curl -s -X POST '$CI_TRIGGER_URL' \
    -H 'Content-type: application/json' \
    -d '{\'branch\': \'$branch\', \'trigger\': \'claude-code\'}'

  # 记录
  echo '$(date -Iseconds) | CI triggered for $branch' >> ~/.claude/ci.log
fi

echo '{'continue': true}'
```

### Hook：推送后触发

```bash
#!/bin/bash
# trigger-deploy.sh

input=$(cat)
command=$(echo '$input' | jq -r '.tool_input.command')

# 检查 git push
if [[ '$command' =~ 'git push' ]]; then
  # 解析推送目标
  remote=$(echo '$command' | grep -oP 'origin \K\S+')

  if [[ '$remote' =~ 'main|master' ]]; then
    # 触发生产部署
    curl -s -X POST '$DEPLOY_TRIGGER_URL' \
      -H 'Content-type: application/json' \
      -d '{'environment': 'production', 'trigger': 'claude-code'}'

    notify-send 'Deploy' 'Production deployment triggered'
  fi
fi

echo '{'continue': true}'
```

### 配置

```json

          },
          {
            'type': 'command',

}
```

## 企业安全考量

### 1. 权限控制

- MCP Server 使用的 API Token 需要最小权限
- Hook 不应该暴露敏感信息

### 2. 审计日志

所有关键操作记录：

```bash
#!/bin/bash
# audit-log.sh

input=$(cat)
timestamp=$(date -Iseconds)
tool=$(echo '$input' | jq -r '.tool_name')
args=$(echo '$input' | jq -c '.tool_input')

echo '$timestamp | $tool | $args' >> ~/.claude/audit.log
echo '{'continue': true}'
```

### 3. 白名单限制

企业策略限制可用 MCP Server：

```json
{
  'allowedMcpServers': [
    { 'serverName': 'gitlab' },
    { 'serverName': 'jira' },
    { 'serverName': 'knowledge' }
  ]
}
```

### 4. Hook 验证

所有关键操作前验证：

```bash
#!/bin/bash
# enterprise-validator.sh

input=$(cat)
tool=$(echo '$input' | jq -r '.tool_name')

# 禁止删除操作
if [[ '$tool' =~ 'delete|remove|destroy' ]]; then
  jq -n '{continue: false, decision: 'block', reason: 'Delete operations forbidden'}'
  exit 0
fi

# 检查权限
user=$(whoami)
if ! grep -q '$user' ~/.claude/allowed-users.txt; then
  jq -n --arg user '$user' \
    '{continue: false, decision: 'block', reason: 'User ' + $user + ' not authorized'}'
  exit 0
fi

echo '{'continue': true}'
```
