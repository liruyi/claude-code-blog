扩展开发完成后，还需要考虑调试、性能优化、安全保障。本篇总结常见问题和解决方案。

## 调试技巧

### 1. 使用 /debug Skill

Claude Code 内置 `/debug` Skill，可以查看当前会话日志：

```
/debug Skill not loading
```

输出：
- Debug log 路径
- 最近 20 行日志
- 错误和警告摘要

### 2. 查看 Hook 执行日志

Hook 执行会写入 debug log：

```bash
# 查看 Hook 日志
tail -f ~/.claude/debug-*.log | grep -E 'Hook|PreToolUse|PostToolUse'
```

### 3. MCP Server 调试

**查看 Server 状态**：

```
/mcp
```

输出每个 Server 的连接状态。

**查看 Server 工具**：

```
/mcp server-name
```

**手动测试 MCP Server**：

```bash
# 启动 Server 并测试
echo '{'jsonrpc': '2.0', 'id': 1, 'method': 'tools/list'}' | node dist/index.js
```

### 4. Skill 调试

**检查 Skill 注册**：

```
/skills
```

**Skill 不生效的原因**：

| 问题 | 解决 |
|------|------|
| frontmatter 格式错误 | 确保 YAML 格式正确 |
| name 有特殊字符 | 只用字母、数字、下划线、连字符 |
| 文件位置错误 | 放在 `~/.claude/skills/` 或 `.claude/skills/` |
| disableModelInvocation=true | 用户必须显式调用 |

### 5. Hook 调试

**验证 Hook 配置**：

```json
// 简单测试 Hook
{
  'hooks': {
    'PreToolUse': [
      {
        'hooks': [
          {
            'type': 'command',
            'command': 'echo 'Hook executed' >> /tmp/hook-test.log'
          }
        ]
      }
    ]
  }
}
```

执行操作后检查 `/tmp/hook-test.log`。

### 6. 日志级别

设置环境变量启用详细日志：

```bash
export CLAUDE_CODE_DEBUG=1
claude
```

## 性能考量

### 1. Hook 执行时间

Hook 会阻塞操作：

```
用户提交命令
     │
     ├─► PreToolUse Hook（阻塞）
     │       │
     │       └─► 如果耗时 5 秒，命令延迟 5 秒
     │
     └─► 工具执行
```

**最佳实践**：
- Hook 尽快返回（< 1 秒）
- 复杂验证用异步服务
- 避免 HTTP 调用阻塞

**解决方案**：使用 async Hook：

```json
{
  'type': 'command',
  'command': 'async-validator.sh',
  'timeout': 5
}
```

Hook 返回 `{async: true}` 时，后续操作不等 Hook 完成。

### 2. MCP Server 启动

stdio MCP Server 每次连接启动一个进程：

**优化**：
- Server 脚本预热（预先编译）
- 避免 `npm install` 在启动时
- 使用 `npx -y` 跳过安装确认

### 3. Skill 内容大小

Skill 内容会注入对话：

```
用户: /skill
     │
     ▼
Skill 内容（可能 2000 tokens）注入对话
```

**最佳实践**：
- Skill 内容精简（< 1000 tokens）
- 避免重复内容
- 用 Resources 提供大文档

### 4. Fork vs Inline

Fork 模式有开销：

| 模式 | 开销 |
|------|------|
| Inline | 无（共享上下文） |
| Fork | 启动子 Agent + 独立上下文 |

**选择**：
- 简单任务用 Inline
- 大量输出用 Fork（避免污染主上下文）

### 5. 多 Hook 并行

同一事件多个 Hook 顺序执行：

```json
{
  'PreToolUse': [
    { 'type': 'command', 'command': 'hook1.sh' },  // 1 秒
    { 'type': 'command', 'command': 'hook2.sh' },  // 1 秒
    { 'type': 'command', 'command': 'hook3.sh' }   // 1 秒
  ]
}
```

总阻塞时间：3 秒。

**优化**：合并 Hook：

```bash
#!/bin/bash
# combined-hook.sh

# 所有检查在一个脚本
check1 && check2 && check3
echo '{'continue': true}'
```

## 安全考量

### 1. Hook 输入验证

Hook 输入来自 stdin，需要验证：

```bash
#!/bin/bash

# 验证 JSON 格式
input=$(cat)
if ! echo '$input' | jq -e . > /dev/null 2>&1; then
  echo '{'continue': true}'  # 无效输入，放行
  exit 0
fi

# 正常处理
tool=$(echo '$input' | jq -r '.tool_name')
...
```

### 2. MCP Server 权限

MCP Server 的 API Token 应最小权限：

| 服务 | 权限建议 |
|------|----------|
| Slack | 只特定频道 |
| GitHub | 只读 + PR review |
| GitLab | 只特定项目 |
| Jira | 只创建任务 |

### 3. Hook 不暴露敏感信息

**错误示例**：

```bash
#!/bin/bash
# ❌ 危险：暴露 Token

input=$(cat)
curl 'https://api.example.com/notify?token=SECRET_TOKEN'
```

**正确做法**：

```bash
#!/bin/bash
# ✅ 安全：环境变量

curl 'https://api.example.com/notify' \
  -H 'Authorization: Bearer $API_TOKEN'
```

### 4. 企业白名单

限制可用的 MCP Server：

```json
{
  'allowedMcpServers': [
    { 'serverName': 'approved-server' }
  ],
  'deniedMcpServers': [
    { 'serverUrl': 'https://external.example.com/*' }
  ]
}
```

### 5. Hook 不能修改工具输入（除非显式）

Hook 默认不能修改工具输入：

```json
// ❌ 无效：直接修改
{
  'continue': true,
  'updatedInput': { 'command': 'new-command' }  // 无效
}
```

需要通过 `hookSpecificOutput`：

```json
// ✅ 有效：通过 hookSpecificOutput
{
  'continue': true,
  'hookSpecificOutput': {
    'hookEventName': 'PreToolUse',
    'updatedInput': { 'command': 'new-command' }
  }
}
```

## 常见踩坑

### 1. Skill 名称冲突

多个 Skill 同名会冲突：

```
~/.claude/skills/review.md
.claude/skills/review.md
```

**解决**：项目 Skill 覆盖用户 Skill，或用不同名称。

### 2. Hook matcher 不匹配

matcher 语法错误：

```json
// ❌ 错误：正则语法
{ 'matcher': 'Bash.*' }

// ✅ 正确：glob 语法
{ 'matcher': 'Bash' }
{ 'matcher': 'Bash|Edit' }
{ 'matcher': 'Bash(git *)' }
```

### 3. MCP Server 环境变量缺失

```json
{
  'mcpServers': {
    'api': {
      'env': {
        'API_KEY': '${API_KEY}'  // 如果 API_KEY 未设置
      }
    }
  }
}
```

**解决**：
- 设置环境变量：`export API_KEY=xxx`
- 或直接写值（不推荐）

### 4. Hook stdout 格式错误

Hook 输出必须是 JSON：

```bash
// ❌ 错误：非 JSON
echo 'approved'

// ✅ 正确：JSON
echo '{'continue': true}'
```

### 5. MCP 工具名前缀

MCP 工具调用时需要前缀：

```
mcp__slack__post_message   // 注意双下划线
mcp__github__create_issue
```

在 Skill 中：

```yaml
allowedTools: [mcp__slack__*]  // 匹配所有 slack 工具
```

### 6. Fork 模式上下文不共享

Fork 的 Skill 不共享主对话历史：

```
用户: 前面我们讨论了 bug A
     │
     ▼
Fork Skill: 我不知道 bug A（独立上下文）
```

**解决**：在 Skill prompt 中传递关键信息。

### 7. PostToolUse 不接收失败

`PostToolUse` 只在成功时触发：

```
工具执行成功 → PostToolUse
工具执行失败 → PostToolUseFailure
```

需要两个 Hook 分别处理。
