当对话历史超过 Token 预算时，必须压缩。但压缩会丢失信息，如何保留关键内容？本篇分析 Claude Code 的压缩策略。

## 压缩的困境

### 信息丢失

压缩必然丢失信息：

```
原始消息（100K tokens）：
- 用户：修复登录 bug
- AI：我来分析...读取 auth.ts...读取 login.ts...编辑 auth.ts...
- 工具：Read auth.ts (500 lines output)
- 工具：Read login.ts (300 lines output)
- 工具：Edit auth.ts (success)
- 用户：测试一下
- ...

压缩后（30K tokens）：
- [摘要] 用户报告登录 bug，AI 分析并修复了 auth.ts 和 login.ts
- [摘要] 测试通过
```

丢失：工具输出详情、中间推理过程。

### 关键信息保留

关键是识别什么是重要的：

| 信息类型 | 是否保留 | 原因 |
|----------|----------|------|
| 用户决策 | 保留 | 用户意图是核心 |
| AI 最终结论 | 保留 | AI 的决策结果重要 |
| 工具成功/失败 | 保留摘要 | 知道执行结果 |
| 工具详细输出 | 不保留 | 通常是噪音 |
| 中间推理 | 不保留 | 最终结论更重要 |
| 文件路径 | 保留 | 知道操作了哪些文件 |

## 四种压缩策略

### 1. Snip：历史剪裁

**最简单**：直接剪裁旧消息。

```typescript
function snipCompact(messages: Message[], keepRecent: number): CompactResult {
  const total = messages.length

  if (total <= keepRecent) {
    return { messages, snipped: 0 }
  }

  const recent = messages.slice(total - keepRecent)
  const snipped = messages.slice(0, total - keepRecent)

  // 插入剪裁摘要
  const summary = generateSnipSummary(snipped)
  recent.unshift({
    type: 'system',
    content: `📝 [已剪裁 ${snipped.length} 条消息]\n${summary}`,
  })

  return { messages: recent, snipped: snipped.length }
}

function generateSnipSummary(messages: Message[]): string {
  // 提取关键信息生成摘要
  const userMessages = messages.filter(m => m.type === 'user')
  const toolCalls = messages.filter(m => m.type === 'tool_use')

  return `用户消息 ${userMessages.length} 条，工具调用 ${toolCalls.length} 次`
}
```

**优点**：
- 实现简单
- 速度快
- 保留最近的完整上下文

**缺点**：
- 可能丢失重要决策
- 线性丢失，不智能

**适用**：快速压缩，不需要精细控制。

### 2. Microcompact：结构保留

**精细**：保留消息结构，精简内容。

```typescript
function microcompact(messages: Message[]): Message[] {
  return messages.map(msg => {
    switch (msg.type) {
      case 'user':
        // 用户消息保留，但精简
        return {
          ...msg,
          content: compactText(msg.content, 1000),  // 精简到 1000 tokens
        }

      case 'assistant':
        // AI 响应保留关键结论
        return {
          ...msg,
          content: extractKeyConclusions(msg.content),
        }

      case 'tool_use':
        // 工具调用保留名称和关键参数
        return {
          type: 'tool_use_summary',
          tool_name: msg.tool_name,
          input: compactToolInput(msg.input),
        }

      case 'tool_result':
        // 工具结果只保留摘要
        return {
          type: 'tool_result_summary',
          success: !msg.is_error,
          summary: summarizeToolOutput(msg.content, 500),
        }

      default:
        return msg
    }
  })
}

function compactToolInput(input: object): object {
  // 只保留关键参数
  const keyParams = ['file_path', 'command', 'pattern']

  return Object.fromEntries(
    Object.entries(input)
      .filter(([key]) => keyParams.includes(key))
  )
}
```

**优点**：
- 保留消息结构
- 保留关键参数
- 可以追溯操作序列

**缺点**：
- 处理复杂
- 需要理解每种消息类型

**适用**：需要保留操作历史的场景。

### 3. Context Collapse：相似合并

**智能**：合并相似的消息组。

```typescript
function contextCollapse(messages: Message[]): Message[] {
  // 识别可合并的消息组
  const groups = identifyCollapsibleGroups(messages)

  const collapsed = []

  for (const group of groups) {
    if (group.canCollapse) {
      collapsed.push(collapseGroup(group))
    } else {
      collapsed.push(...group.messages)
    }
  }

  return collapsed
}

function identifyCollapsibleGroups(messages: Message[]): MessageGroup[] {
  const groups: MessageGroup[] = []
  let currentGroup: MessageGroup = { type: 'mixed', messages: [], canCollapse: false }

  for (const msg of messages) {
    // 检测连续的同类消息
    if (isReadMessage(msg) && currentGroup.type === 'read') {
      currentGroup.messages.push(msg)
    } else if (isGrepMessage(msg) && currentGroup.type === 'grep') {
      currentGroup.messages.push(msg)
    } else {
      // 类型变化，保存当前组，开始新组
      if (currentGroup.messages.length > 0) {
        groups.push(currentGroup)
      }
      currentGroup = {
        type: getGroupType(msg),
        messages: [msg],
        canCollapse: canCollapseType(msg),
      }
    }
  }

  if (currentGroup.messages.length > 0) {
    groups.push(currentGroup)
  }

  return groups
}

function collapseGroup(group: MessageGroup): Message {
  switch (group.type) {
    case 'read':
      return {
        type: 'collapsed_read',
        files: group.messages.map(m => m.input.file_path),
        summary: `读取了 ${group.messages.length} 个文件`,
      }

    case 'grep':
      return {
        type: 'collapsed_grep',
        patterns: group.messages.map(m => m.input.pattern),
        summary: `搜索了 ${group.messages.length} 个模式`,
      }

    default:
      // 无法合并，返回第一条（其他丢弃）
      return group.messages[0]
  }
}
```

**优点**：
- 大幅减少 token（合并同类）
- 保留操作概览
- 识别模式（如'AI 先读了 10 个文件再编辑'）

**缺点**：
- 需要识别可合并组
- 可能合并不该合并的

**适用**：大量同类操作的场景。

### 4. Autocompact：自动选择

**智能**：根据场景自动选择策略。

```typescript
function autocompact(messages: Message[]): CompactResult {
  const analysis = analyzeMessages(messages)

  // 根据分析选择策略
  if (analysis.hasLongToolOutput) {
    // 工具输出大 → Context Collapse 合并
    return contextCollapse(messages)
  }

  if (analysis.hasRecentImportantDecision) {
    // 最近有重要决策 → Snip 保留最近
    return snipCompact(messages, 30)
  }

  if (analysis.hasDiverseOperations) {
    // 操作多样 → Microcompact 保留结构
    return microcompact(messages)
  }

  // 默认：Snip
  return snipCompact(messages, 50)
}

function analyzeMessages(messages: Message[]): MessageAnalysis {
  const recentMessages = messages.slice(-10)

  return {
    hasLongToolOutput: messages.some(m =>
      m.type === 'tool_result' && estimateTokens(m.content) > 10000
    ),

    hasRecentImportantDecision: recentMessages.some(m =>
      m.type === 'assistant' && m.content.includes('决定') || m.content.includes('decided')
    ),

    hasDiverseOperations: countUniqueTools(messages) > 5,
  }
}
```

**优点**：
- 自动适应场景
- 综合各策略优点

**缺点**：
- 分析开销
- 可能选择错误

**适用**：默认压缩策略。

## Reactive Compact：413 错误触发

当 API 返回 413 时紧急压缩：

```typescript
async function handle413Error(messages: Message[]): Promise<Message[]> {
  // 1. 尝试 Autocompact
  const compacted = autocompact(messages)

  // 2. 重新估算 token
  const tokens = estimateTokens(compacted)

  // 3. 如果仍超限，更激进压缩
  if (tokens > CONTEXT_LIMIT) {
    // 只保留用户消息和最终结论
    const minimal = preserveEssential(compacted)
    return minimal
  }

  return compacted
}

function preserveEssential(messages: Message[]): Message[] {
  // 只保留：
  // 1. 最近的用户消息（5 条）
  // 2. AI 的最终结论（最近 1 条）
  // 3. 重要决策摘要

  const userMessages = messages.filter(m => m.type === 'user').slice(-5)
  const assistantMessages = messages.filter(m => m.type === 'assistant').slice(-1)

  // 生成摘要
  const summary = generateSessionSummary(messages)

  return [
    { type: 'system', content: `📝 会话摘要：${summary}` },
    ...userMessages,
    ...assistantMessages,
  ]
}
```

## 压缩时机

### 预防性压缩

```typescript
const PRE_COMPACT_THRESHOLD = 150000  // 150K

// 每次请求前检查
function checkPreCompact(messages: Message[]): Message[] {
  const tokens = estimateTokens(messages)

  if (tokens > PRE_COMPACT_THRESHOLD) {
    console.log('Token 超限，自动压缩...')
    return autocompact(messages)
  }

  return messages
}
```

### 响应性压缩

```typescript
// API 返回 413 时
async function handleRequest(messages: Message[]) {
  try {
    return await apiRequest(messages)
  } catch (error) {
    if (error.code === 413) {
      console.log('收到 413 错误，紧急压缩...')
      const compacted = handle413Error(messages)
      return await apiRequest(compacted)
    }
    throw error
  }
}
```

## 压缩摘要生成

压缩后生成摘要，帮助 AI 理解历史：

```typescript
function generateSessionSummary(messages: Message[]): string {
  const userIntents = extractUserIntents(messages)
  const aiDecisions = extractAIDecisions(messages)
  const toolsUsed = summarizeToolsUsed(messages)

  return `
本次会话：
- 用户目标：${userIntents.join('、')}
- AI 决策：${aiDecisions.join('、')}
- 使用工具：${toolsUsed}

关键文件：${extractKeyFiles(messages).join('、')}
  `
```

## 压缩原则总结

### 1. 优先级保留

```
用户消息 > AI 决策 > 工具摘要 > 工具详情
```

### 2. 策略选择

```
大量同类操作 → Context Collapse
多样操作 → Microcompact
快速压缩 → Snip
默认 → Autocompact
```

### 3. 预防优于响应

```
150K 时预防性压缩
避免等待 413 错误
```

### 4. 摘要生成

```
压缩后生成摘要
帮助 AI 理解历史
```

### 5. 关键信息识别

```
用户意图、AI 决策、文件路径、操作结果
```
