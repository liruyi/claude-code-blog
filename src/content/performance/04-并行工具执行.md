AI 可以同时执行多个工具。但不是所有工具都能并行——有些会冲突。本篇分析并发安全的判断与执行策略。

## 并行的必要性

### 传统顺序执行

```
工具1 → 完成 → 工具2 → 完成 → 工具3 → 完成

时间：t1 + t2 + t3
```

### 并行执行

```
工具1 ───►
工具2 ───► ───► 完成
工具3 ───►

时间：max(t1, t2, t3)
```

**效果**：如果有 3 个 Read 操作，每个 1 秒：
- 顺序：3 秒
- 并行：1 秒

## 并发安全判断

### 工具声明

每个工具必须声明是否并发安全：

```typescript
interface Tool {
  // 核心方法
  name: string
  call(input, context): Promise<ToolResult>

  // 并发安全声明
  isConcurrencySafe(input: object): boolean

  // 只读声明
  isReadOnly(input: object): boolean

  // 破坏性声明
  isDestructive?(input: object): boolean
}
```

### 默认值

**默认假设不安全**：

```typescript
const TOOL_DEFAULTS = {
  isConcurrencySafe: (_input) => false,  // 默认不安全
  isReadOnly: (_input) => false,          // 默认写入
  isDestructive: (_input) => false,       // 默认不破坏
}
```

工具必须**显式声明安全**。

### 各工具的声明

| 工具 | isConcurrencySafe | 原因 |
|------|-------------------|------|
| Read | ✅ true | 只读取，不互相影响 |
| Glob | ✅ true | 只搜索，不修改 |
| Grep | ✅ true | 只搜索，不修改 |
| WebFetch | ✅ true | 独立网络请求 |
| WebSearch | ✅ true | 独立网络请求 |
| Edit | ❌ false | 可能编辑同一文件 |
| Write | ❌ false | 可能写入同一文件 |
| Bash | ❌ false | 命令可能互相影响 |
| Agent | ❌ false | 子 Agent 可能共享状态 |
| Skill | ❌ false | 可能修改上下文 |

### 输入相关判断

某些工具根据输入判断：

```typescript
// Edit：如果编辑不同文件，可以并行
class EditTool {
  isConcurrencySafe(input: EditInput): boolean {
    // 如果两个 Edit 编辑同一个文件，不安全
    // 但我们无法在工具层面检测，所以默认不安全
    return false
  }
}

// Bash：某些命令可以并行
class BashTool {
  isConcurrencySafe(input: BashInput): boolean {
    // 简单判断：cd 命令不安全（影响后续命令）
    // git 命令相对安全（不同操作）
    const command = input.command

    if (command.startsWith('cd ')) {
      return false  // cd 改变目录状态
    }

    if (command.includes('&&') || command.includes(';')) {
      return false  // 复合命令可能有依赖
    }

    // 默认不安全
    return false
  }
}
```

## 并行执行策略

### 分组执行

```typescript
async function executeTools(toolCalls: ToolCall[], context: ToolContext): ToolResult[] {
  // 分组：并发安全 vs 不安全
  const safeCalls = toolCalls.filter(c => c.tool.isConcurrencySafe(c.input))
  const unsafeCalls = toolCalls.filter(c => !c.tool.isConcurrencySafe(c.input))

  // 安全的并行执行
  const safeResults = await Promise.all(
    safeCalls.map(c => executeSingleTool(c, context))
  )

  // 不安全的顺序执行
  const unsafeResults = []
  for (const c of unsafeCalls) {
    const result = await executeSingleTool(c, context)
    unsafeResults.push(result)

    // 更新上下文（可能影响后续工具）
    if (result.contextModifier) {
      context = result.contextModifier(context)
    }
  }

  return [...safeResults, ...unsafeResults]
}
```

### 依赖检测

更精细的依赖检测：

```typescript
function detectDependencies(toolCalls: ToolCall[]): DependencyGraph {
  // 检测工具间的依赖

  // 1. 文件依赖
  // Read(file1) 和 Edit(file1) 不能并行
  const fileDeps = detectFileDependencies(toolCalls)

  // 2. 命令依赖
  // Bash(cd dir) 和 Bash(git status) 有依赖
  const cmdDeps = detectCommandDependencies(toolCalls)

  // 3. 上下文依赖
  // Skill 和后续工具有依赖
  const contextDeps = detectContextDependencies(toolCalls)

  return { fileDeps, cmdDeps, contextDeps }
}

function detectFileDependencies(toolCalls: ToolCall[]): Map<string, string[]> {
  const deps = new Map<string, string[]>()

  // 收集文件操作
  const fileOps = toolCalls.filter(c =>
    c.tool.name === 'Read' || c.tool.name === 'Edit' || c.tool.name === 'Write'
  )

  // 检查是否有相同文件
  for (const op1 of fileOps) {
    for (const op2 of fileOps) {
      if (op1 !== op2 && op1.input.file_path === op2.input.file_path) {
        // 同文件操作，有依赖
        deps.set(op1.id, [...deps.get(op1.id) || [], op2.id])
      }
    }
  }

  return deps
}
```

### 智能调度

基于依赖图调度：

```typescript
function scheduleExecution(toolCalls: ToolCall[], deps: DependencyGraph): ExecutionPlan {
  // 构建执行计划
  // 无依赖的并行，有依赖的顺序

  const plan: ExecutionPlan = []
  const executed = new Set<string>()
  const pending = [...toolCalls]

  while (pending.length > 0) {
    // 找出无依赖的工具
    const ready = pending.filter(c => {
      const dependencies = deps.get(c.id) || []
      return dependencies.every(d => executed.has(d))
    })

    if (ready.length === 0) {
      // 所有待执行都有依赖，顺序执行第一个
      ready.push(pending[0])
    }

    // 这批可以并行执行
    plan.push(ready)

    // 标记已执行
    for (const c of ready) {
      executed.add(c.id)
      pending.splice(pending.indexOf(c), 1)
    }
  }

  return plan
}
```

## Streaming Tool Executor 的并发

### 流式并发

在 API 流式返回时，工具可以提前开始：

```typescript
class StreamingToolExecutor {
  private executions: Map<string, Promise<ToolResult>>

  // 添加工具（立即开始执行）
  addTool(toolCall: ToolCall) {
    if (toolCall.tool.isConcurrencySafe(toolCall.input)) {
      // 安全工具立即开始
      const promise = this.executeTool(toolCall)
      this.executions.set(toolCall.id, promise)
    } else {
      // 不安全工具排队等待
      this.queue.push(toolCall)
    }
  }

  // 获取已完成结果
  getCompletedResults(): ToolResult[] {
    const completed = []

    for (const [id, promise] of this.executions) {
      // 使用 Promise.race 检查状态
      const state = Promise.race([
        promise.then(r => ({ status: 'done', result: r })),
        Promise.resolve({ status: 'pending' })
      ])

      if (state.status === 'done') {
        completed.push(state.result)
        this.executions.delete(id)
      }
    }

    return completed
  }
}
```

### 效果图

```
API 流式返回：
事件1: ToolUse(Read file1) → 立即开始 Read
事件2: ToolUse(Read file2) → 立即开始 Read（并行）
事件3: ToolUse(Read file3) → 立即开始 Read（并行）
事件4: ToolUse(Edit file1) → 排队等待（不安全）
...
API 完成 → 执行排队的 Edit
```

## 并发限制

### 最大并发数

```typescript
const MAX_CONCURRENT_TOOLS = 10

async function executeParallel(toolCalls: ToolCall[]): Promise<ToolResult[]> {
  // 限制并发数
  const batches = chunk(toolCalls, MAX_CONCURRENT_TOOLS)

  const results = []

  for (const batch of batches) {
    const batchResults = await Promise.all(
      batch.map(c => executeTool(c))
    )
    results.push(...batchResults)
  }

  return results
}
```

### 资源限制

```typescript
// 某些工具消耗大量资源，限制并发
const RESOURCE_INTENSIVE_TOOLS = ['Agent', 'WebSearch']

function adjustConcurrency(toolCalls: ToolCall[]): number {
  const intensiveCount = toolCalls.filter(c =>
    RESOURCE_INTENSIVE_TOOLS.includes(c.tool.name)
  ).length

  // 如果有资源密集工具，降低并发
  if (intensiveCount > 0) {
    return Math.min(3, MAX_CONCURRENT_TOOLS - intensiveCount)
  }

  return MAX_CONCURRENT_TOOLS
}
```

## 并发安全原则

### 1. 默认不安全

```
工具必须显式声明 isConcurrencySafe: true
```

### 2. 按输入判断

```
某些工具根据输入判断安全
```

### 3. 分组执行

```
安全工具并行 + 不安全工具顺序
```

### 4. 依赖检测

```
同文件操作有依赖
```

### 5. 资源限制

```
资源密集工具降低并发
```
