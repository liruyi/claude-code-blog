会话数据需要持久化，但读写不能影响性能。本篇分析 JSONL 格式的存储优化策略。

## 存储的挑战

### 数据量

```
典型会话：
- 50 条消息
- 20 个工具执行结果
- 文件内容缓存
- Token 数：100K-200K

存储大小：5-20MB
```

### 性能要求

```
写入：
- 消息实时追加
- 不能阻塞 UI
- 写入延迟 < 10ms

读取：
- 启动时快速加载
- 恢复会话时快速读取
- 加载时间 < 100ms
```

## JSONL 格式选择

### 为什么 JSONL？

```jsonl
{'type':'user','content':'Hello','timestamp':'2024-01-15T10:00:00Z'}
{'type':'assistant','content':'Hi!','timestamp':'2024-01-15T10:00:05Z'}
{'type':'tool_result','tool':'Read','output':'...','timestamp':'2024-01-15T10:00:10Z'}
```

**优势**：
- **追加写入**：每条消息独立一行，直接追加
- **并发安全**：不需要重写整个文件
- **快速读取尾部**：只需要读取最后 N 行
- **容错性好**：单行损坏不影响其他行

### 对比 JSON 数组

```json
[
  {'type':'user','content':'Hello'},
  {'type':'assistant','content':'Hi!'},
  {'type':'tool_result','tool':'Read','output':'...'}
]
```

**问题**：
- 写入需要重写整个文件
- 大文件写入慢
- 读取需要解析整个数组

## 追加写入优化

### 基本写入

```typescript
async function appendMessage(sessionId: string, message: Message): void {
  const filePath = getSessionPath(sessionId)
  const line = JSON.stringify(message) + '\n'

  // 直接追加
  await fs.appendFile(filePath, line, 'utf-8')
}
```

### 批量写入

```typescript
const writeQueue: Message[] = []
let writePending = false

async function queueMessage(message: Message): void {
  writeQueue.push(message)

  if (!writePending) {
    writePending = true
    // 延迟 50ms 批量写入
    setTimeout(() => flushWriteQueue(), 50)
  }
}

async function flushWriteQueue(): void {
  if (writeQueue.length === 0) {
    writePending = false
    return
  }

  const batch = writeQueue.splice(0, writeQueue.length)
  const lines = batch.map(m => JSON.stringify(m) + '\n').join('')

  await fs.appendFile(sessionPath, lines, 'utf-8')
  writePending = false
}
```

### 写入性能对比

```
❌ 单条写入
每次消息 → 立即写入 → 100 次写入 → 总耗时 500ms

✅ 批量写入
消息进入队列 → 50ms 后批量写入 → 5 次写入 → 总耗时 50ms
```

## 快速尾部读取

### 只读最近消息

```typescript
async function readRecentMessages(sessionId: string, count: number = 50): Message[] {
  const filePath = getSessionPath(sessionId)

  // 获取文件大小
  const stats = await fs.stat(filePath)
  const fileSize = stats.size

  // 从尾部读取
  const chunkSize = Math.min(fileSize, 64 * 1024)  // 最多读 64KB
  const start = Math.max(0, fileSize - chunkSize)

  const buffer = Buffer.alloc(chunkSize)
  const fd = await fs.open(filePath, 'r')
  await fd.read(buffer, 0, chunkSize, start)
  await fd.close()

  // 解析行
  const lines = buffer.toString('utf-8').split('\n')

  // 取最后 count 行
  const recentLines = lines.slice(-count)

  return recentLines
    .filter(line => line.trim())
    .map(line => JSON.parse(line))
}
```

### 智能估算

```typescript
async function estimateLinesToRead(sessionId: string, targetCount: number): number {
  // 平均每行大小（历史统计）
  const avgLineSize = 500  // bytes

  // 需要读取的字节数
  const targetBytes = avgLineSize * targetCount

  // 实际读取更多（确保足够）
  const readBytes = targetBytes * 2

  return Math.min(readBytes, 64 * 1024)
}
```

### 启动只加载摘要

```typescript
async function loadSessionList(): SessionSummary[] {
  const summaryFiles = await glob('~/.claude/sessions/*.summary.json')

  // 并行加载摘要
  const summaries = await Promise.all(
    summaryFiles.map(f => fs.readFile(f, 'utf-8').then(JSON.parse))
  )

  return summaries.sort((a, b) =>
    new Date(b.lastActiveAt).getTime() - new Date(a.lastActiveAt).getTime()
  )
}
```

### 恢复时按需加载

```typescript
async function resumeSession(sessionId: string): Session {
  // 1. 加载摘要
  const summary = await loadSummary(sessionId)

  // 2. 显示摘要给用户
  displaySummary(summary)

  // 3. 用户确认后加载完整历史
  if (userConfirmResume()) {
    const messages = await readRecentMessages(sessionId, 100)
    return { summary, messages }
  }

  return { summary, messages: [] }
}
```

## Tombstone 删除

### 删除不重写

```typescript
async function deleteMessage(sessionId: string, messageIndex: number): void {
  // 不删除行，而是标记 Tombstone
  const tombstone = {
    type: 'tombstone',
    deletedIndex: messageIndex,
    timestamp: Date.now(),
  }

  await appendMessage(sessionId, tombstone)
}
```

### 读取时过滤

```typescript
async function readMessages(sessionId: string): Message[] {
  const lines = await readAllLines(sessionId)

  // 收集 Tombstone
  const deletedIndices = new Set<number>()
  for (const line of lines) {
    const msg = JSON.parse(line)
    if (msg.type === 'tombstone') {
      deletedIndices.add(msg.deletedIndex)
    }
  }

  // 过滤已删除
  return lines
    .filter(line => {
      const msg = JSON.parse(line)
      return msg.type !== 'tombstone' && !deletedIndices.has(msg.index)
    })
    .map(JSON.parse)
}
```

### Tombstone 重写限制

```typescript
const TOMBSTONE_THRESHOLD = 100  // 最多 100 个 Tombstone

async function checkTombstoneCount(sessionId: string): void {
  const count = await countTombstones(sessionId)

  if (count > TOMBSTONE_THRESHOLD) {
    // 重写文件，清理 Tombstone
    const messages = await readMessages(sessionId)
    await rewriteSession(sessionId, messages)
  }
}

async function rewriteSession(sessionId: string, messages: Message[]): void {
  // 重写整个文件
  const lines = messages.map(m => JSON.stringify(m) + '\n').join('')
  await fs.writeFile(getSessionPath(sessionId), lines, 'utf-8')
}
```

## 并发安全

### 文件锁

```typescript
import { lock } from 'proper-lockfile'

async function safeAppend(sessionId: string, message: Message): void {
  const filePath = getSessionPath(sessionId)

  // 获取文件锁
  const release = await lock(filePath)

  try {
    await fs.appendFile(filePath, JSON.stringify(message) + '\n', 'utf-8')
  } finally {
    await release()
  }
}
```

## 压缩存储

### 大消息压缩

```typescript
async function compressMessage(message: Message): Message {
  if (message.type === 'tool_result' && message.output.length > 10000) {
    // 大输出压缩
    const compressed = await gzip(message.output)

    return {
      ...message,
      output: compressed.toString('base64'),
      compressed: true,
    }
  }

  return message
}

async function decompressMessage(message: Message): Message {
  if (message.compressed) {
    const decompressed = await gunzip(Buffer.from(message.output, 'base64'))
    return {
      ...message,
      output: decompressed.toString('utf-8'),
      compressed: false,
    }
  }

  return message
}
```

### 历史消息压缩

```typescript
async function compressOldMessages(sessionId: string): void {
  const messages = await readMessages(sessionId)

  // 只保留最近 50 条完整消息
  const recent = messages.slice(-50)
  const old = messages.slice(0, -50)

  // 压缩旧消息
  const compressedOld = await Promise.all(
    old.map(m => compressMessage(m))
  )

  // 重写文件
  await rewriteSession(sessionId, [...compressedOld, ...recent])
}
```

```
✅ 有优化

```

## 存储原则总结

### 1. JSONL 格式

```
每行一条消息 → 追加写入 → 不重写
```

### 2. 批量写入

```
消息队列 → 50ms 批量 → 减少写入次数
```

### 3. 尾部读取

```
只读最后 N 行 → 64KB chunk → 快速恢复
```

### 4. 摘要优先

```
启动加载摘要 → 恢复时按需加载完整
```

### 5. Tombstone 删除

```
标记删除 → 不重写 → 限制 100 个后清理
```
