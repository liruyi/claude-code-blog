文件读取是最常用的操作之一。每次 Read 都消耗 Token。本篇分析如何避免重复发送已读文件。

## 缓存的必要性

### 问题场景

```
对话开始：
AI: Read src/auth.ts → 发送 5000 tokens
用户: 修改登录逻辑
AI: Read src/auth.ts → 又发送 5000 tokens（浪费！）
```

### Token 成本

```
文件大小：10KB
Token 数：约 2500 tokens
每次请求成本：$0.003 (输入)
重复发送 3 次：浪费 $0.009 + 占用上下文
```

## 文件状态追踪

### 状态结构

```typescript
type FileState = {
  content: string        // 文件完整内容
  timestamp: number      // 读取时的 mtime
  hash?: string          // 内容 hash（可选）
  readCount: number      // 读取次数
  lastReadAt: number     // 最后读取时间
}

// 全局文件状态缓存
const readFileState: Map<string, FileState> = new Map()
```

### 读取时记录

```typescript
async function readFile(path: string, context: ToolContext): ToolResult {
  // 获取文件 stats
  const stats = await fs.stat(path)
  const currentMtime = stats.mtimeMs

  // 检查缓存
  const cachedState = readFileState.get(path)

  if (cachedState) {
    // 检查是否修改
    if (cachedState.timestamp === currentMtime) {
      // 文件未修改，不重复发送
      return {
        type: 'file_unchanged',
        path,
        cached: true,
        message: 'File unchanged since last read'
      }
    }

    // 文件已修改，重新读取
    console.log(`文件已修改，重新读取: ${path}`)
  }

  // 实际读取文件
  const content = await fs.readFile(path, 'utf-8')

  // 更新缓存
  readFileState.set(path, {
    content,
    timestamp: currentMtime,
    readCount: (cachedState?.readCount || 0) + 1,
    lastReadAt: Date.now(),
  })

  return {
    type: 'file_content',
    path,
    content,
  }
}
```

## 修改时间检测

### mtime 检查

```typescript
async function checkFileModified(path: string): boolean {
  const cachedState = readFileState.get(path)

  if (!cachedState) {
    return true  // 无缓存，视为修改
  }

  const stats = await fs.stat(path)
  const currentMtime = stats.mtimeMs

  // mtime 相同 = 未修改
  if (currentMtime === cachedState.timestamp) {
    return false
  }

  // mtime 变化，检查内容是否真的变了
  const currentContent = await fs.readFile(path, 'utf-8')

  // 内容对比
  if (currentContent === cachedState.content) {
    // 云同步、杀毒软件等可能触发 mtime 变化但内容不变
    // 更新缓存时间戳（避免下次再检查）
    readFileState.set(path, {
      ...cachedState,
      timestamp: currentMtime,
    })
    return false
  }

  return true  // 内容确实变了
}
```

### 为什么检查内容？

```
场景1：云同步
Dropbox 同步 → 文件 mtime 变化 → 内容不变
如果只看 mtime → 误判为修改 → 重复发送

场景2：Git checkout
git checkout → 文件恢复 → mtime 变化 → 内容可能相同
需要实际对比

场景3：杀毒软件扫描
扫描文件 → mtime 变化 → 内容不变
```

## 增量读取

### 范围读取缓存

```typescript
type RangeCache = {
  ranges: Array<{
    offset: number
    limit: number
    content: string
  }>
  fullContent?: string  // 如果完整读取
  timestamp: number
}

const rangeCache: Map<string, RangeCache> = new Map()

async function readFileRange(path: string, offset: number, limit: number): ToolResult {
  const cache = rangeCache.get(path)

  // 检查已有范围是否覆盖
  if (cache) {
    const covered = cache.ranges.find(r =>
      r.offset <= offset && r.offset + r.limit >= offset + limit
    )

    if (covered && cache.timestamp === getMtime(path)) {
      // 从缓存切片
      const content = covered.content.slice(
        offset - covered.offset,
        offset - covered.offset + limit
      )

      return {
        type: 'file_range_cached',
        content,
        cached: true,
      }
    }
  }

  // 实际读取范围
  const content = await readFilePartial(path, offset, limit)

  // 更新范围缓存
  if (!cache) {
    rangeCache.set(path, {
      ranges: [{ offset, limit, content }],
      timestamp: getMtime(path),
    })
  } else {
    cache.ranges.push({ offset, limit, content })
  }

  return { type: 'file_range', content }
}
```

### 合并范围

```typescript
function optimizeRanges(ranges: Range[]): Range[] {
  // 检查是否覆盖整个文件
  const totalSize = getFileSize()

  const covered = ranges.reduce((sum, r) => sum + r.limit, 0)

  if (covered >= totalSize) {
    // 已经覆盖整个文件，合并为完整读取
    const fullContent = mergeRanges(ranges)

    return [{
      offset: 0,
      limit: totalSize,
      content: fullContent,
    }]
  }

  return ranges
}
```

## Edit 后更新缓存

### 编辑时同步更新

```typescript
async function editFile(input: EditInput, context: ToolContext): ToolResult {
  const path = input.file_path
  const cachedState = readFileState.get(path)

  // 执行编辑
  const newContent = applyEdit(cachedState.content, input.old_string, input.new_string)

  // 写入文件
  await fs.writeFile(path, newContent)

  // 更新缓存（关键！）
  readFileState.set(path, {
    content: newContent,
    timestamp: Date.now(),  // 更新时间戳
    readCount: cachedState.readCount,
    lastReadAt: cachedState.lastReadAt,
  })

  return { type: 'edit_success' }
}
```

### 不更新缓存的后果

```
❌ 不更新缓存
Edit auth.ts → 文件内容变化
下次 Read auth.ts → mtime 检测到变化 → 重新读取 → 重新发送

✅ 更新缓存
Edit auth.ts → 同时更新缓存
下次 Read auth.ts → mtime 匹配 → 使用缓存 → 不发送
```

### 内存限制

```typescript
const MAX_CACHE_SIZE = 100  // 最多缓存 100 个文件

function checkCacheSize() {
  if (readFileState.size > MAX_CACHE_SIZE) {
    // 清理最旧的缓存
    const entries = [...readFileState.entries()]
      .sort((a, b) => a[1].lastReadAt - b[1].lastReadAt)

    // 删除最旧的 20 个
    for (let i = 0; i < 20; i++) {
      readFileState.delete(entries[i][0])
    }
  }
}
```

### 大文件特殊处理

```typescript
const LARGE_FILE_THRESHOLD = 50000  // 50KB

function cacheLargeFile(path: string, content: string) {
  if (content.length > LARGE_FILE_THRESHOLD) {
    // 大文件只缓存摘要
    readFileState.set(path, {
      content: '',  // 不缓存完整内容
      summary: summarizeContent(content),  // 缓存摘要
      timestamp: getMtime(path),
      isLarge: true,
    })
  } else {
    // 小文件完整缓存
    readFileState.set(path, {
      content,
      timestamp: getMtime(path),
    })
  }
}
```

## 缓存原则总结

### 1. 每次读取都缓存

```
Read → 记录 content + timestamp
```

### 2. mtime 检查

```
下次 Read → 对比 mtime → 相同则跳过
```

### 3. 内容二次验证

```
mtime 变化 → 对比内容 → 确保真的修改
```

### 4. Edit 后同步更新

```
Edit → 更新缓存 content + timestamp
```

### 5. 增量读取合并

```
范围读取 → 检查覆盖 → 合并为完整
```
