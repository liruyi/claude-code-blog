本系列分析了 Claude Code 的性能优化策略。最后，我们提炼十大 AI Agent 性能原则。

## 原则 1：流式优先原则

**永远不要等待完成再显示**

### 流式响应

```
❌ 传统模式
请求 ───────► [等待 10s] ───────► 显示完整响应

✅ 流式模式
请求 ───► 开始显示 ───► 更新 ───► 更新 ───► 完成
        0.5s          2s        5s        10s
```

### 设计对策

```typescript
async function* query(options): AsyncGenerator<Message> {
  for await (const event of streamApiRequest(options)) {
    yield processEvent(event)  // 立即 yield
  }
}
```

### 实现要点

- AsyncGenerator 模式
- 边接收边处理
- 更新频率 100ms

## 原则 2：并行最大化原则

**能并行的一定并行**

### 并行机会

```
初始化并行：
- 配置加载
- MCP 连接
- 遥测初始化

工具执行并行：
- Read file1
- Read file2
- Read file3  （并发安全）
```

### 设计对策

```typescript
// 初始化并行
await Promise.all([
  initConfig(),
  initMCP(),
  initTelemetry(),
])

// 工具并行
const safeTools = toolCalls.filter(t => t.tool.isConcurrencySafe())
await Promise.all(safeTools.map(t => t.execute()))
```

### 实现要点

- 识别可并行操作
- 工具声明并发安全
- 使用 Promise.all

## 原则 3：Token 珍惜原则

**每个 Token 都要优化**

### Token 来源

```
系统提示：10K-30K
工具定义：5K-10K
用户消息：5K-20K
工具输出：可能 50K-200K（最大）
```

### 设计对策

```
工具输出控制：
- 截断到 50K
- 保留关键信息
- 智能摘要

系统提示优化：
- 工具定义精简
- 动态工具列表
```

### 实现要点

- 工具输出截断
- 系统提示精简
- Token 预算分配

## 原则 4：压缩智能原则

**压缩要保留关键信息**

### 压缩策略

```
保留：
- 用户意图
- AI 决策
- 操作摘要
- 文件路径

丢弃：
- 工具详细输出
- 中间推理过程
```

### 设计对策

```typescript
function smartCompact(messages): Message[] {
  return messages.map(m => {
    if (m.type === 'tool_result') {
      // 只保留摘要
      return { summary: summarize(m.output) }
    }
    return m
  })
}
```

### 实现要点

- 识别关键信息
- 多种压缩策略
- 生成压缩摘要

## 原则 5：缓存有效原则

**避免重复发送**

### 缓存机会

```
文件缓存：
- 已读文件内容
- 文件修改时间
- 相同范围不重复发送

会话缓存：
- MCP 连接保持
- 配置缓存
```

### 设计对策

```typescript
const readFileState = new Map<string, FileState>()

async function readFile(path: string) {
  const state = readFileState.get(path)
  if (state && !fileChanged(path)) {
    return { type: 'file_unchanged' }  // 不发送内容
  }

  const content = await fs.readFile(path)
  readFileState.set(path, { content, mtime: getMtime(path) })
  return { type: 'file_content', content }
}
```

### 实现要点

- 文件状态追踪
- 修改时间检测
- 会话级缓存

## 原则 6：延迟隐藏原则

**预连接、推测执行**

### 延迟隐藏

```
API 预连接：
- 启动时建立 TCP + TLS
- 首次请求延迟减少 150ms

推测执行：
- 对话框显示时运行分类器
- 高置信度允许自动消失
```

### 设计对策

```typescript
// 启动时预连接
async function preconnectAnthropicApi() {
  await connect('api.anthropic.com', 443)
}

// 对话框时推测执行
function showDialog() {
  void classifyCommand(command)  // 不阻塞
}
```

### 实现要点

- 启动预连接关键服务
- 推测执行不阻塞
- 结果可能丢弃

## 原则 7：上下文预算原则

**主动管理不超限**

### 预算管理

```
预防性：
- 超过 150K 自动压缩
- 不等待 413 错误

响应性：
- 413 时紧急压缩
- 只保留关键消息
```

### 设计对策

```typescript
function checkTokenBudget(messages): Message[] {
  const tokens = estimateTokens(messages)

  if (tokens > 150000) {
    return autocompact(messages)
  }

  return messages
}
```

### 实现要点

- 定期检查 token
- 预防性压缩
- 响应性压缩

## 原则 8：增量处理原则

**不一次性处理全部**

### 增量机会

```
会话加载：
- 不加载完整历史
- 只加载摘要

文件读取：
- 大文件分块读取
- 范围读取而非全读
```

### 设计对策

```typescript
async function loadSession(sessionId) {
  // 只加载摘要
  const summary = await readSessionSummary(sessionId)

  // 按需加载历史
  if (needFullHistory) {
    const history = await readFullHistory(sessionId)
  }

  return summary
}
```

### 实现要点

- 摘要优先加载
- 按需完整加载
- 大文件分块

## 原则 9：隔离执行原则

**大任务用 Fork**

### Fork 优势

```
Fork 模式：
- 独立 token 预算
- 不污染主上下文
- 可并行执行

适用：
- 大量搜索任务
- 复杂研究任务
- 不需要用户干预
```

### 设计对策

```markdown
---
context: fork
agent: Explore
---

# 研究任务
...
```

### 实现要点

- Fork 模式选择
- 独立上下文
- 完成后通知

## 原则 10：测量驱动原则

**优化要有数据支撑**

### 测量指标

```
启动时间：100ms
首字节时间：500ms
工具执行时间：按工具
Token 效率：输入/输出比
压缩效率：压缩后/压缩前
```

### 设计对策

```typescript
// 测量关键指标
const metrics = {
  startupTime: measureStartup(),
  firstByteTime: measureFirstByte(),
  toolExecutionTime: measureToolExecution(),
  tokenEfficiency: calculateTokenEfficiency(),
}

// 记录到遥测
logMetrics(metrics)
```

### 实现要点

- 关键指标定义
- 定期测量
- 数据驱动优化

## 原则总结表

| 原则 | 核心问题 | 设计对策 |
|------|----------|----------|
| 1 流式优先 | 如何减少等待？ | AsyncGenerator |
| 2 并行最大化 | 如何加速？ | Promise.all |
| 3 Token 珍惜 | 如何省钱？ | 输出截断 + 精简 |
| 4 压缩智能 | 如何保留信息？ | 关键信息保留 |
| 5 缓存有效 | 如何避免重复？ | 文件状态追踪 |
| 6 延迟隐藏 | 如何减少感知延迟？ | 预连接 + 推测 |
| 7 上下文预算 | 如何不超限？ | 预防性压缩 |
| 8 增量处理 | 如何处理大数据？ | 摘要 + 分块 |
| 9 隔离执行 | 如何处理大任务？ | Fork 模式 |
| 10 测量驱动 | 如何知道优化效果？ | 指标测量 |

## 应用这些原则

当你优化 AI Agent 性能时：

1. **测量当前状态**：先有数据
2. **识别瓶颈**：找到最慢的部分
3. **应用原则**：针对性优化
4. **测量效果**：验证改进
