CLI 工具的启动速度直接影响用户体验。如果每次调用都要等待几秒，用户会非常不满。本篇分析 Claude Code 如何实现 100ms 冷启动。

## 启动速度的重要性

用户期望 CLI 快速响应：

| 操作 | 期望时间 |
|------|----------|
| `--version` | < 50ms |
| `--help` | < 100ms |
| 交互模式启动 | < 500ms |
| 首次响应 | < 1s |

Claude Code 是复杂应用：
- React/Ink UI
- 30+ 工具模块
- MCP 连接
- 配置加载

如果全部加载，启动可能需要 5-10s。

## 快速路径设计

Claude Code 在入口文件 (`cli.tsx`) 实现快速路径：

### 版本输出（零导入）

```typescript
// cli.tsx 顶部，任何导入之前

// 版本输出 -- 零导入！
if (args.length === 1 && (args[0] === '--version' || args[0] === '-v')) {
  console.log(`${MACRO.VERSION} (Claude Code)`)
  return
}

// 帮助输出 -- 零导入！
if (args[0] === '--help' || args[0] === '-h') {
  console.log(HELP_TEXT)  // 内联字符串
  return
}
```

**关键**：这些检查在任何模块导入之前执行。

**效果**：
- `claude --version` → 50ms（只解析 args）
- 不加载 React、Ink、任何工具

### 特殊命令的条件导入

```typescript
// Daemon Worker -- 保持轻量
if (feature('DAEMON') && args[0] === '--daemon-worker') {
  // 只导入需要的模块
  await import('./daemon-worker.js').then(m => m.run(args[1]))
  return
}

// Bridge 模式 -- 需要完整流程
if (feature('BRIDGE_MODE') && args[0] === 'remote-control') {
  enableConfigs()
  // Auth 检查
  // GrowthBook gate 检查
  // Policy limits 检查
  await bridgeMain(args.slice(1))
  return
}
```

**关键**：使用动态 `import()` 而不是静态导入。

### Bun 特性标志

`feature('DAEMON')` 是 Bun 的编译时特性标志：

```typescript
function feature(name: string): boolean {
  // Bun 在构建时内联结果
  // 如果 DAEMON 未启用，代码被完全删除（DCE）
  return bunFeatures[name] ?? false
}
```

**效果**：未启用的特性代码不会出现在最终产物中。

## 延迟导入策略

### 主入口的延迟加载

```typescript
// cli.tsx

// 快速路径检查...
if (handledQuickPath) return

// 现在才开始导入主模块
const { init } = await import('./init.js')
const { main } = await import('./main.js')

await init()
await main(args)
```

**效果**：快速路径用户不需要等待主模块加载。

### 工具的延迟注册

工具不预先加载，按需注册：

```typescript
// 不好的做法：
import { BashTool } from './tools/BashTool.js'
import { ReadTool } from './tools/ReadTool.js'
// ... 30+ 导入

// Claude Code 的做法：
const tools: Tool[] = []

function registerTool(name: string) {
  // 按需导入
  const tool = require(`./tools/${name}.js`)
  tools.push(tool)
}

// 只在需要时注册
if (needsBashTool) registerTool('BashTool')
```

### Schema 的延迟构建

Zod schema 构建有开销：

```typescript
// 不好的做法：
const inputSchema = z.object({
  command: z.string(),
  timeout: z.number().optional(),
})

// Claude Code 的做法：
const inputSchema = lazySchema(() => z.object({
  command: z.string(),
  timeout: z.number().optional(),
}))

// 只有在验证时才构建 schema
function lazySchema(builder: () => ZodSchema) {
  let schema: ZodSchema | undefined
  return {
    parse: (input) => {
      schema ??= builder()  // 延迟构建
      return schema.parse(input)
    }
  }
}
```

## 初始化流程优化

### Memoize 单次执行

初始化只执行一次：

```typescript
export const init = memoize(async (): Promise<void> => {
  enableConfigs()
  applySafeConfigEnvironmentVariables()
  setupGracefulShutdown()
  initialize1PEventLogging()
  configureGlobalMTLS()
  configureGlobalAgents()
  preconnectAnthropicApi()
}))
```

**效果**：多次调用 `init()` 只执行一次。

### 配置分阶段应用

敏感配置分两阶段：

```typescript
// 阶段 1：安全配置（信任对话框之前）
applySafeConfigEnvironmentVariables()  // 只应用安全变量

// 阶段 2：完整配置（用户确认信任后）
applyFullConfigEnvironmentVariables()
```

**安全变量示例**：

```typescript
const SAFE_CONFIG_VARS = new Set([
  'CLAUDE_CODE_DEBUG',
  'CLAUDE_CODE_DISABLE_TELEMETRY',
])
```

### API 预连接

减少首次请求延迟：

```typescript
// 在初始化时预连接
async function preconnectAnthropicApi() {
  // 建立 TCP + TLS 连接（约 100-200ms）
  // 但因为它与其他初始化并行，用户感受不到

  const socket = await connect({
    host: 'api.anthropic.com',
    port: 443,
  })

  // 连接保持，首次请求时使用
  preconnectedSocket = socket
}
```

**效果**：首次 API 请求延迟减少 150ms。

## Fire-and-Forget 初始化

不阻塞主流程的初始化：

```typescript
// 遥测初始化 -- 不阻塞
void Promise.all([
  import('./services/analytics/firstPartyEventLogger.js'),
  import('./services/analytics/growthbook.js'),
]).then(([fp, gb]) => {
  fp.initialize1PEventLogging()
  gb.onGrowthBookRefresh(() => {
    void fp.reinitialize1PEventLoggingIfConfigChanged()
  })
})
```

**关键**：`void` 前缀表示不等待结果。

**效果**：遥测初始化不影响用户感知的启动时间。

## 启动优化原则总结

### 1. 快速路径优先

```typescript
// 任何导入之前
if (isQuickPath(args)) {
  handleQuickPath(args)
  return
}
```

### 2. 动态导入

```typescript
// 不用静态导入
import { heavyModule } from './heavy.js'  // ❌

// 用动态导入
await import('./heavy.js')  // ✅
```

### 3. 延迟构建

```typescript
// 不要在模块加载时构建
const schema = z.object({ ... })  // ❌

// 延迟构建
const schema = lazySchema(() => z.object({ ... }))  // ✅
```

### 4. 并行初始化

```typescript
// 不要串行
await initAnalytics()
await initMCP()
await initHooks()  // ❌

// 并行
await Promise.all([
  initAnalytics(),
  initMCP(),
  initHooks(),
])  // ✅
```

### 5. 预连接关键服务

```typescript
// 预连接 API
preconnectAnthropicApi()  // 减少 TLS 时间
```

### 6. Memoize 单次操作

```typescript
// 只执行一次
export const init = memoize(async () => { ... })
```
