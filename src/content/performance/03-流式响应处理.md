API 响应是 AI Agent 最大的延迟源。如果等待完整响应才开始处理，用户体验很差。本篇分析 Claude Code 如何实现流式响应处理。

## 为什么流式重要？

### 传统请求模式

```
用户发送请求 ─────► API 处理 ─────► 返回完整响应 ─────► 开始显示

时间线：
[等待 10 秒空白]
└─────────────────────────────────────────────────────────┘
                                                          └─► 显示响应
```

**问题**：用户看到空白 10 秒。

### 流式响应模式

```
用户发送请求 ─────► API 开始流式返回 ─────► 边接收边显示

时间线：
[开始显示]──► [更新]──► [更新]──► [更新]──► [完成]
0.5s         1s        3s        7s        10s
```

**效果**：用户立即看到响应，感知延迟降低。

## AsyncGenerator 设计

Claude Code 使用 AsyncGenerator 实现流式：

### query 函数签名

```typescript
async function* query({
  messages,
  tools,
  context,
}): AsyncGenerator<Message, void, unknown> {
  // ...
}
```

**关键特性**：
- `async function*`：异步生成器
- `yield`：每次 yield 一个消息
- `for await`：消费者可以流式接收

### 生成器实现

```typescript
async function* query(options: QueryOptions): AsyncGenerator<Message> {
  // 主循环
  while (true) {
    // 准备请求
    const requestParams = prepareRequestParams(options)

    // 流式调用 API
    for await (const event of streamApiRequest(requestParams)) {
      // 处理不同事件类型

      // 文本块开始
      if (event.type === 'content_block_start') {
        yield { type: 'text_start', block_id: event.index }
      }

      // 文本增量
      if (event.type === 'content_block_delta' && event.delta.type === 'text_delta') {
        yield {
          type: 'text_delta',
          block_id: event.index,
          text: event.delta.text,
        }
      }

      // 工具调用开始
      if (event.type === 'content_block_start' && event.content_block.type === 'tool_use') {
        yield {
          type: 'tool_start',
          tool_name: event.content_block.name,
          tool_id: event.content_block.id,
        }
      }

      // 工具调用增量（参数）
      if (event.type === 'content_block_delta' && event.delta.type === 'input_json_delta') {
        yield {
          type: 'tool_input_delta',
          tool_id: event.index,
          partial_json: event.delta.partial_json,
        }
      }

      // 消息停止
      if (event.type === 'message_stop') {
        // 处理工具调用
        const toolResults = await executeTools(message.toolUses)

        // yield 工具结果
        for (const result of toolResults) {
          yield { type: 'tool_result', result }
        }
      }
    }

    // 决定继续或退出
    if (shouldContinue()) {
      options.messages = updatedMessages
    } else {
      return { type: 'completed' }
    }
  }
}
```

### 消费者使用

```typescript
// REPL 中消费 query
async function processUserInput(prompt: string) {
  const queryIterator = query({
    messages: [{ role: 'user', content: prompt }],
    tools: availableTools,
    context: toolContext,
  })

  for await (const message of queryIterator) {
    // 根据消息类型更新 UI
    switch (message.type) {
      case 'text_start':
        startTextBlock(message.block_id)
        break

      case 'text_delta':
        appendText(message.block_id, message.text)
        break

      case 'tool_start':
        showToolStart(message.tool_name)
        break

      case 'tool_result':
        showToolResult(message.result)
        break
    }

    // 用户可以随时中断
    if (interrupted) {
      queryIterator.return()  // 结束生成器
      break
    }
  }
}
```

## Streaming Tool Executor

核心创新：**在 API 流式返回时就开始执行工具**

### 问题

传统流程：

```
API 开始返回 ───► ... ───► API 完成 ───► 解析工具 ───► 执行工具 ───► 返回结果

等待时间：API 时间 + 工具执行时间
```

如果 API 10 秒，工具执行 5 秒，总共 15 秒。

### 解决方案

```typescript
class StreamingToolExecutor {
  private toolExecutions: Map<string, Promise<ToolResult>>
  private completedResults: ToolResult[] = []

  // API 流式返回时添加工具
  addTool(toolBlock: PartialToolBlock) {
    // 工具参数可能还在传输中
    // 等待完整参数后开始执行

    const executionPromise = this.waitForCompleteInput(toolBlock)
      .then(input => this.executeTool(toolBlock.name, input))

    this.toolExecutions.set(toolBlock.id, executionPromise)
  }

  // 获取已完成结果
  getCompletedResults(): ToolResult[] {
    // 检查已完成的执行
    for (const [id, promise] of this.toolExecutions) {
      if (promise.status === 'fulfilled') {
        this.completedResults.push(promise.value)
        this.toolExecutions.delete(id)
      }
    }

    return [...this.completedResults]
  }

  // 等待完整输入
  private async waitForCompleteInput(toolBlock: PartialToolBlock): Promise<object> {
    // 监听后续 delta，拼接完整 JSON
    let partialJson = toolBlock.partial_json

    while (!isCompleteJson(partialJson)) {
      // 等待下一个 delta
      const delta = await nextDelta(toolBlock.id)
      partialJson += delta.partial_json
    }

    return JSON.parse(partialJson)
  }

  // 执行工具
  private async executeTool(name: string, input: object): Promise<ToolResult> {
    const tool = getTool(name)
    return await tool.call(input, this.context)
  }
}
```

### 效果

```
API 开始返回 ───► 发现工具 ───► 开始执行 ───► ... ───► API 完成 ───► 获取结果

并行时间：max(API 时间, 工具执行时间)
```

如果 API 10 秒，工具执行 5 秒，总共 max(10, 5) = 10 秒。

### 实现

```typescript
async function* query(options: QueryOptions): AsyncGenerator<Message> {
  const streamingExecutor = new StreamingToolExecutor()

  for await (const event of streamApiRequest(options)) {
    // 发现工具调用，立即添加
    if (event.type === 'content_block_start' && event.content_block.type === 'tool_use') {
      streamingExecutor.addTool({
        id: event.content_block.id,
        name: event.content_block.name,
        partial_json: '',
      })
    }

    // 工具参数增量
    if (event.type === 'content_block_delta' && event.delta.type === 'input_json_delta') {
      streamingExecutor.updateToolInput(
        event.index,
        event.delta.partial_json
      )
    }

    // 定期获取已完成结果
    for (const result of streamingExecutor.getCompletedResults()) {
      yield { type: 'tool_result', result }
    }

    // 文本增量直接 yield
    if (event.type === 'content_block_delta' && event.delta.type === 'text_delta') {
      yield { type: 'text_delta', text: event.delta.text }
    }
  }

  // 最终获取所有结果
  const remainingResults = streamingExecutor.getRemainingResults()
  for (const result of remainingResults) {
    yield { type: 'tool_result', result }
  }
}
```

## 并发工具执行

并发安全的工具可以并行执行：

### 并发安全判断

```typescript
interface Tool {
  // 工具必须声明是否并发安全
  isConcurrencySafe(input: object): boolean
}
```

**示例**：

```typescript
// Read 工具：并发安全（读取不互相影响）
class ReadTool {
  isConcurrencySafe() { return true }
}

// Edit 工具：不并发安全（编辑同一文件可能冲突）
class EditTool {
  isConcurrencySafe() { return false }
}

// Bash 工具：不并发安全（命令可能互相影响）
class BashTool {
  isConcurrencySafe() { return false }
}
```

### 执行策略

```typescript
async function executeTools(toolCalls: ToolCall[]): Promise<ToolResult[]> {
  // 分组：并发安全 vs 不安全
  const safeTools = toolCalls.filter(t => t.tool.isConcurrencySafe(t.input))
  const unsafeTools = toolCalls.filter(t => !t.tool.isConcurrencySafe(t.input))

  // 并发安全的并行执行
  const safeResults = await Promise.all(
    safeTools.map(t => t.tool.call(t.input))
  )

  // 不安全的顺序执行
  const unsafeResults = []
  for (const t of unsafeTools) {
    const result = await t.tool.call(t.input)
    unsafeResults.push(result)
  }

  return [...safeResults, ...unsafeResults]
}
```

### 效果

```
顺序执行（传统）：
Read file1 (1s) ───► Read file2 (1s) ───► Read file3 (1s) ───► 完成
总计：3s

并行执行：
Read file1 (1s) ───►
Read file2 (1s) ───► ───► 完成
Read file3 (1s) ───►
总计：1s
```

## 中断处理

AsyncGenerator 支持优雅中断：

### AbortController

```typescript
async function* query(options: QueryOptions): AsyncGenerator<Message> {
  const abortSignal = options.abortSignal

  // 定期检查中断
  while (!abortSignal.aborted) {
    // 正常处理...

    if (abortSignal.aborted) {
      // 中断时清理
      yield { type: 'cancelled', reason: 'User interrupted' }
      return
    }
  }
}
```

### 外部中断

```typescript
// 用户 Ctrl+C
const abortController = new AbortController()

process.on('SIGINT', () => {
  abortController.abort()
})

// query 传入 abortSignal
const iterator = query({
  messages,
  abortSignal: abortController.signal,
})

// 消费时检查
for await (const msg of iterator) {
  if (msg.type === 'cancelled') {
    console.log('已中断')
    break
  }
}
```

## 流式显示优化

### 更新频率控制

终端刷新率有限，需要控制更新频率：

```typescript
// 文本更新节流
let lastTextUpdate = 0
const TEXT_UPDATE_INTERVAL = 100  // 100ms

function appendText(blockId: string, text: string) {
  const now = Date.now()
  if (now - lastTextUpdate < TEXT_UPDATE_INTERVAL) {
    // 缓存，稍后更新
    pendingText[blockId] += text
    return
  }

  lastTextUpdate = now
  // 实际渲染
  renderText(blockId, pendingText[blockId])
  pendingText[blockId] = ''
}
```

### 增量渲染

只渲染变化部分：

```typescript
// 不好的做法：全量渲染
function renderMessage(message: Message) {
  clearScreen()
  renderAllMessages(history)  // 重新渲染所有
}

// 好的做法：增量渲染
function appendTextDelta(blockId: string, delta: string) {
  // 只追加新内容
  const block = getBlock(blockId)
  block.content += delta
  renderBlock(block)  // 只渲染这个块
}
```

## 流式设计原则

### 1. 立即开始显示

```
❌ 等待完整响应
API 完成 → 显示

✅ 流式显示
API 开始 → 显示 → 更新 → 更新 → 完成
```

### 2. 工具提前执行

```
❌ 顺序执行
API 完成 → 解析工具 → 执行工具

✅ 流式执行
API 开始 → 发现工具 → 开始执行 → API 完成 → 获取结果
```

### 3. 并发最大化

```
❌ 所有工具顺序执行
Read → Read → Read → 完成

✅ 安全工具并行执行
Read ───►
Read ───► ───► 完成
Read ───►
```

### 4. 支持中断

```
❌ 无法中断
请求开始 → 必须等待完成

✅ 随时可中断
请求开始 → 用户 Ctrl+C → 立即停止
```

### 5. 增量渲染

```
❌ 全量渲染
每次更新重绘所有

✅ 增量渲染
只更新变化部分
```
