AI CLI 的执行可能很长，用户随时可能中断。如何优雅处理中断，不丢失已完成的工作？本篇分析 Claude Code 的错误与中断处理设计。

## 中断的必要性

用户中断的原因：

| 原因 | 说明 |
|------|------|
| 发现方向错误 | AI 的理解偏离预期 |
| 等待太长 | 任务执行超过预期时间 |
| 需要调整 | 想修改输入或指令 |
| 紧急离开 | 有其他任务需要处理 |

## Ctrl+C 的处理流程

### 中断信号捕获

```typescript
const abortController = new AbortController()

process.on('SIGINT', () => {
  // 1. 设置中断标志
  abortController.abort()

  // 2. 显示中断提示
  console.log('\n^C 已中断')

  // 3. 保存当前状态
  saveSessionState()

  // 4. 提示恢复方法
  console.log('使用 /resume 继续或 /save 保存')
})
```

### AbortController 传递

中断信号传递给所有异步操作：

```typescript
async function query(options: QueryOptions): AsyncGenerator<Message> {
  const signal = options.abortSignal

  while (!signal?.aborted) {
    // API 调用传递信号
    for await (const event of streamApiRequest(params, signal)) {
      if (signal?.aborted) {
        yield { type: 'cancelled', reason: 'User interrupted' }
        return
      }

      yield processEvent(event)
    }
  }

  if (signal?.aborted) {
    yield { type: 'cancelled' }
  }
}
```

### 工具执行中断

每个工具都要处理中断：

```typescript
async function bashTool(input, context, signal) {
  if (signal?.aborted) {
    return { type: 'cancelled' }
  }

  // 启动子进程
  const child = spawn(input.command, { signal })

  // 监听中断
  signal?.addEventListener('abort', () => {
    child.kill('SIGTERM')
  })

  // 等待完成或中断
  try {
    const result = await child.promise
    return result
  } catch (e) {
    if (signal?.aborted) {
      return { type: 'cancelled' }
    }
    throw e
  }
}
```

## 中断后的状态保存

### 保存什么

| 内容 | 是否保存 | 原因 |
|------|----------|------|
| 已完成的消息 | 保存 | 用户可能需要 |
| 已执行的工具结果 | 保存摘要 | 避免重新执行 |
| 进行中的工具 | 不保存 | 状态不完整 |
| 用户输入 | 保存 | 恢复时需要 |

### 状态保存实现

```typescript
function saveSessionState() {
  const state = {
    // 已完成的消息
    completedMessages: messages.filter(m => m.status === 'completed'),

    // 工具执行摘要
    toolSummaries: toolResults.map(r => ({
      tool: r.tool_name,
      success: r.success,
      summary: summarizeOutput(r.output),
    })),

    // 当前输入
    currentInput: inputBox.value,

    // 时间戳
    interruptedAt: new Date().toISOString(),
  }

  // 写入会话文件
  writeSessionFile(sessionId, state)
}
```

## 恢复机制

### /resume 命令

恢复中断的会话：

```
/resume

恢复会话 session-abc...

上次中断时：
- 已读取 src/auth.ts
- 已编辑 src/api.ts (部分)
- 正在执行 npm test (未完成)

继续对话：
> 继续之前的任务
```

### 恢复流程

```typescript
async function resumeSession(sessionId: string) {
  // 1. 加载保存的状态
  const state = readSessionFile(sessionId)

  // 2. 显示上次进度
  displayProgress(state.toolSummaries)

  // 3. 提示用户
  console.log('上次任务：', state.currentInput)
  console.log('已完成：', state.toolSummaries.length, '个操作')

  // 4. 等待用户输入
  const input = await readUserInput()

  // 5. 继续执行
  await query({ messages: state.completedMessages, input })
}
```

## 错误处理

### 错误类型

| 类型 | 来源 | 处理方式 |
|------|------|----------|
| 输入错误 | 用户 | 提示修正 |
| 工具执行失败 | 工具 | 解释原因 + 建议 |
| API 错误 | Anthropic | 简化提示 + 重试 |
| 网络错误 | 网络 | 显示状态 + 重试 |
| 权限错误 | 权限系统 | 解释 + 提供选项 |

### 错误消息设计

**原则：可理解、可解决**

```
❌ 技术性错误
Error: ENOENT: no such file or directory, open 'src/auth.ts'
errno: -2, code: 'ENOENT', syscall: 'open'

✅ 用户友好错误
找不到文件：src/auth.ts

可能的原因：
1. 文件路径不正确
2. 文件不存在

建议：
- 使用 Glob 查找相似文件：glob src/**/*.ts
- 检查当前目录：pwd
```

### API 错误重试

```typescript
async function apiRequestWithRetry(params, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await apiRequest(params)
    } catch (error) {
      if (error.code === 429) {  // Rate limit
        const waitTime = error.headers?.['retry-after'] || 60
        console.log(`速率限制，等待 ${waitTime} 秒后重试...`)
        await sleep(waitTime * 1000)
        continue
      }

      if (error.code === 413) {  // Payload too large
        console.log('内容过大，正在压缩...')
        params.messages = await compactMessages(params.messages)
        continue
      }

      if (error.code >= 500) {  // Server error
        console.log(`服务器错误，重试 ${i + 1}/${maxRetries}...`)
        await sleep(1000)
        continue
      }

      // 其他错误不重试
      throw error
    }
  }

  throw new Error('重试次数已达上限')
}
```

## 错误日志

### 调试日志

关键错误记录到调试日志：

```typescript
function logError(error: Error, context: ErrorContext) {
  const logEntry = {
    timestamp: new Date().toISOString(),
    error: {
      type: error.constructor.name,
      message: error.message,
      stack: error.stack,
    },
    context: {
      tool: context.toolName,
      input: sanitizeInput(context.input),
      sessionId: context.sessionId,
    },
  }

  // 写入调试日志
  appendDebugLog(logEntry)
}

// 敏感信息过滤
function sanitizeInput(input: object): object {
  const sanitized = { ...input }

  // 移除敏感字段
  const sensitiveFields = ['password', 'token', 'apiKey', 'secret']
  for (const field of sensitiveFields) {
    if (sanitized[field]) {
      sanitized[field] = '[REDACTED]'
    }
  }

  return sanitized
}
```

## 设计原则

### 1. 中断不丢失工作

```
Ctrl+C → 保存状态 → 提示恢复 → 不丢失已完成的工作
```

### 2. 错误可理解

```
技术错误 → 翻译为用户语言 → 提供解决建议
```

### 3. 提供恢复选项

```
失败后：重试 / 查看 / 跳过 / 修改
```

### 4. 重试智能

```
速率限制 → 等待后重试
内容过大 → 压缩后重试
服务器错误 → 简单重试
```

### 5. 日志可调试

```
关键错误记录
用户可查看
敏感信息过滤
```
