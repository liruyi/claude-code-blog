REPL（Read-Eval-Print Loop）是 Claude Code 的主界面。它既是对话界面，也是工具执行界面，还是状态监控界面。本篇分析这个复杂界面的设计。

## REPL 的多重角色

传统 REPL（如 Python）：

```
>>> 输入
输出
>>> 输入
输出
```

简单、线性。

AI REPL：

```
用户输入 ──────► AI 思考 ──────► 工具执行 ──────► 结果 ──────► 继续思考 ──────► 输出
     │              │              │              │              │
     │              │              │              │              │
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  输入框         思考动画        工具进度       结果摘要       消息显示
```

多重状态、并行处理。

### 四大区域

| 区域 | 功能 | 设计挑战 |
|------|------|----------|
| 消息区域 | 显示对话历史 | 长消息如何截断？折叠如何设计？ |
| 工具执行区域 | 显示执行进度 | 多工具如何排列？进度如何更新？ |
| 状态栏 | 显示全局状态 | 信息如何精简？ |
| 输入框 | 用户输入 | 如何支持多行？如何显示提示？ |

## 消息显示设计

### 消息类型

Claude Code 有多种消息类型：

| 类型 | 显示方式 | 用户关注度 |
|------|----------|------------|
| UserMessage | 用户头像 + 内容 | 高（用户自己的输入） |
| AssistantMessage | AI 头像 + 内容 | 高（AI 的响应） |
| ToolUseMessage | 工具图标 + 参数 | 中（好奇 AI 在做什么） |
| ToolResultMessage | 结果摘要 | 低（除非失败） |
| SystemMessage | 系统提示 | 低（通常是配置） |

### 消息折叠

长消息需要折叠：

```
不折叠：
│ │ Tool: [Bash] npm test ──────────────────────────────── │ │
│ │ Result:                                                │ │
│ │ PASS src/auth.test.ts                                  │ │
│ │ PASS src/api.test.ts                                   │ │
│ │ PASS src/utils.test.ts                                 │ │
│ │ ... (500 行)                                           │ │

折叠：
│ │ Tool: [Bash] npm test ✓                                │ │
│ │ Result: 150 tests passed, 2s [展开查看详情]            │ │
```

**折叠策略**：
- 默认折叠工具结果（> 10 行）
- 默认展开 AI 响应（< 100 行）
- 用户可以手动展开/折叠

### 消息渲染优先级

终端刷新率有限，消息渲染要有优先级：

| 优先级 | 消息类型 | 策略 |
|--------|----------|------|
| 最高 | 输入框、状态栏 | 每帧更新 |
| 高 | AI 流式响应 | 每 100ms 更新 |
| 中 | 工具执行进度 | 每 500ms 更新 |
| 低 | 历史消息 | 滚动时更新 |

## 工具执行区域

### 工具状态显示

每个工具显示状态：

```
⏳ 等待执行（等待其他工具完成）
🔄 正在执行（显示进度）
✓ 执行成功
✗ 执行失败
```

### 进度显示

长工具需要进度：

```
短工具（< 1s）：
✓ Read: src/auth.ts

长工具（> 1s）：
🔄 Bash: npm test ──────────────────────── 45%
```

**进度来源**：
- Bash：解析输出中的进度指示
- WebFetch：无法显示进度
- Read：文件大小估算

### 多工具排列

并发执行时，多个工具同时显示：

```
并行执行：
🔄 Bash: npm test ─────────── 45%
🔄 Bash: npm run lint ──────── 30%
⏳ Read: src/auth.ts (waiting)

顺序执行：
✓ Bash: npm test (done)
🔄 Bash: npm run lint ──────── 60%
```

## 状态栏设计

状态栏信息精简但关键：

```
┌────────────────────────────────────────────────────────┐
│ Claude Code │ Opus 4.6 │ session-abc │ tokens: 50K     │
└────────────────────────────────────────────────────────┘
```

### 显示内容

| 信息 | 显示时机 | 格式 |
|------|----------|------|
| 模型名 | 常驻 | `Opus 4.6` |
| Effort | 用户设置时 | `with high effort` |
| Session ID | 常驻（缩短） | `sess-abc` |
| Token 计数 | 常驻 | `50K` |
| 权限模式 | 非 auto 时 | `plan mode` |

### 状态变化动画

状态变化时有微妙动画：

```
AI 思考时：
│ Claude Code ○ with high effort │  (spinner)

AI 输出时：
│ Claude Code ● │                 (solid dot)

工具执行时：
│ Claude Code ⟳ │                 (rotating)
```

## 输入框设计

### 多行输入

支持多行输入：

```
单行：
> 重构 auth.ts

多行：
> 重构 auth.ts
  具体要求：
  - 拆分认证逻辑
  - 提取 token 管理
```

**Enter 行为**：
- 单行：Enter 发送
- 多行：Enter 新行，Ctrl+Enter 发送

### 输入提示

显示提示帮助用户：

```
> 请描述你想要做的事情
> _                                    (光标位置)

可用工具提示：
> /commit, /review, /debug...
```

### 输入历史

上下箭头浏览历史：

```
↑：上一条输入
↓：下一条输入
```

## Ink 渲染技术

Claude Code 使用 Ink（React for CLI）：

### 优势

| 特性 | 说明 |
|------|------|
| React 模型 | 组件化、状态管理 |
| 响应式布局 | 自动适配终端宽度 |
| 流式更新 | 不阻塞主线程 |
| ANSI 支持 | 颜色、样式、动画 |

### 关键组件

```
src/components/
├── MessageDisplay.js      # 消息渲染
├── ToolExecution.js       # 工具进度
├── StatusBar.js           # 状态栏
├── InputBox.js            # 输入框
├── PermissionDialog.js    # 权限对话框
└── Spinner.js             # 加载动画
```

### 渲染性能

Ink 渲染策略：
- 使用 React reconciler
- 只更新变化的部分
- 避免全屏重绘
