会话中断后，用户需要恢复。本篇分析 `/resume` 的 UX 设计。

## 恢复场景

### 用户需求

```
场景1：意外中断
Ctrl+C 中断 → 想继续之前任务

场景2：切换任务
任务 A 中断 → 处理任务 B → 回到任务 A

场景3：重启应用
Claude Code 关闭 → 重新打开 → 继续上次

场景4：跨设备
办公室电脑 → 家里电脑 → 继续同一会话
```

### 恢复挑战

```
挑战1：状态丢失
中断时状态如何保存？

挑战2：上下文恢复
如何快速恢复对话上下文？

挑战3：进度提示
用户如何知道上次进度？

挑战4：分支选择
用户如何选择恢复哪个会话？
```

async function displaySessionList(): void {
  const sessions = await listSessions()

  // 按时间排序
  sessions.sort((a, b) =>
    new Date(b.lastActiveAt).getTime() - new Date(a.lastActiveAt).getTime()
  )

  console.log('\n可用会话:')
  console.log('─'.repeat(50))

  for (let i = 0; i < sessions.length; i++) {

    console.log(`${i + 1}. ${statusIcon} ${session.title}`)
    console.log(`   ${formatTime(session.lastActiveAt)} | ${session.cwd}`)
    console.log(`   ${session.messageCount} 条消息 | ${session.tokenCount} tokens`)
    console.log(`   涉及文件: ${session.filesEdited.length} 编辑, ${session.filesRead.length} 读取`)
  }

  console.log('─'.repeat(50))
  console.log('输入编号恢复，或 Enter 开始新会话')
}

function getStatusIcon(status: string): string {
  switch (status) {
    case 'interrupted': return '⚡'
    case 'completed': return '✓'
    default: return '●'
  }
}

## 恢复流程

### /resume 命令

```typescript
async function resumeCommand(args: string[]): void {
  // 无参数：显示列表
  if (args.length === 0) {
    await interactiveResume()
    return
  }

  // 有参数：直接恢复
  const sessionId = args[0]
  await resumeSession(sessionId)
}

async function interactiveResume(): void {
  // 1. 显示会话列表
  await displaySessionList()

  // 2. 等待用户选择
  const input = await readUserInput()

  if (input.trim() === '') {
    // Enter：开始新会话
    return
  }

  const index = parseInt(input)
  const sessions = await listSessions()

  if (index >= 1 && index <= sessions.length) {
    // 选择有效
    await resumeSession(sessions[index - 1].id)
  } else {
    // 选择无效
    console.log('无效选择')
    await interactiveResume()
  }
}
```

### 恢复会话

```typescript
async function resumeSession(sessionId: string): void {
  // 1. 加载会话摘要
  const metadata = await loadMetadata(sessionId)

  // 2. 显示恢复信息
  displayResumeInfo(metadata)

  // 3. 询问恢复方式
  const resumeMode = await askResumeMode(metadata)

  switch (resumeMode) {
    case 'full':
      // 完整恢复
      await fullResume(metadata)
      break

    case 'summary':
      // 摘要恢复
      await summaryResume(metadata)
      break

    case 'fresh':
      // 从摘要开始新对话
      await freshStart(metadata)
      break
  }
}
```

### 恢复方式选择

```typescript
async function askResumeMode(metadata: SessionMetadata): ResumeMode {
  console.log('\n恢复方式:')
  console.log('  1. 完整恢复（加载所有历史）')
  console.log('  2. 摘要恢复（只加载摘要，节省 token）')
  console.log('  3. 从摘要开始新对话')

  if (metadata.tokenCount > 100000) {
    console.log('\n⚠️ 会话较大，建议摘要恢复')
  }

  const input = await readUserInput()
  const choice = parseInt(input)

  switch (choice) {
    case 1: return 'full'
    case 2: return 'summary'
    case 3: return 'fresh'
    default: return 'summary'
  }
}
```

### 文件状态恢复

```typescript
async function restoreFileStates(sessionId: string): void {
  // 恢复已读文件状态
  const fileStates = await loadFileStates(sessionId)

  for (const [path, state] of Object.entries(fileStates)) {
    // 检查文件是否修改
    const currentMtime = await getMtime(path)

    if (currentMtime > state.timestamp) {
      // 文件已修改，标记需要重新读取
      console.log(`⚠️ ${path} 已被外部修改`)
      readFileState.set(path, {
        ...state,
        needsRefresh: true,
      })
    } else {
      // 文件未修改，恢复缓存
      readFileState.set(path, state)
    }
  }
}
```

### 历史浏览

```typescript
async function historyCommand(args: string[]): void {
  const messages = await loadSessionHistory(currentSession.id)

  // 分页显示
  const pageSize = 20
  let page = 0

  while (true) {
    const start = page * pageSize
    const end = start + pageSize
    const pageMessages = messages.slice(start, end)

    displayHistoryPage(pageMessages, page, messages.length)

    const input = await readUserInput()

    if (input === 'n' && end < messages.length) {
      page++
    } else if (input === 'p' && page > 0) {
      page--
    } else if (input === 'q') {
      break
    }
  }
}

function displayHistoryPage(messages: Message[], page: number, total: number): void {
  console.log(`\n历史 (${page * 20 + 1}-${page * 20 + messages.length}/${total}):`)
  console.log('─'.repeat(50))

  messages.forEach((msg, i) => {
    const role = msg.role === 'user' ? '用户' : 'AI'
    const preview = truncate(msg.content, 100)

    console.log(`${role}: ${preview}`)
  })

  console.log('─'.repeat(50))
  console.log('n=下一页 p=上一页 q=退出')
}
```

async function checkAutoResume(): void {
  // 检查是否有中断的会话
  const interrupted = await findInterruptedSessions()

  if (interrupted.length > 0) {
    // 最近中断的会话
    const latest = interrupted[0]

    console.log('\n发现中断的会话:')
    console.log(`${latest.title} (${formatTime(latest.lastActiveAt)})`)
    console.log('是否恢复? [Y/n]')

    const input = await readUserInput()

    if (input.toLowerCase() !== 'n') {
      await resumeSession(latest.id)
      return
    }
  }

  // 不恢复，开始新会话
  startNewSession()
}

async function branchResume(sessionId: string): void {
  // 从某会话创建分支
  const metadata = await loadMetadata(sessionId)

  console.log('\n创建分支:')
  console.log(`源会话: ${metadata.title}`)

  // 复制会话状态
  const branchSession = {
    id: generateSessionId(),
    parentSessionId: sessionId,
    createdAt: new Date().toISOString(),
    cwd: metadata.cwd,
    messages: [],  // 新消息
    fileStates: await loadFileStates(sessionId),
  }

  // 保存分支关系
  await saveBranchRelation(branchSession.id, sessionId)

  // 开始分支会话
  startSession(branchSession)
}

## 恢复 UX 原则总结

### 1. 自动保存

```
每次消息后自动保存，中断时标记状态
```

### 2. 清晰列表

```
会话列表显示关键信息：标题、时间、状态、文件
```

### 3. 多种恢复方式

```
完整恢复、摘要恢复、从摘要开始新对话
```

### 4. 状态提示

```
显示上次活动、编辑文件、对话摘要
```

### 5. 文件状态检查

```
恢复时检查文件是否被外部修改
```

### 6. 自动提示

```
启动时提示恢复中断的会话
```
