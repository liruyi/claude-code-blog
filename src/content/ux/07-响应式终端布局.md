终端窗口大小不一。本篇分析 Claude Code 的响应式布局设计。

## 响应式的挑战

### 终端宽度差异

```
全屏终端：200+ 列
标准终端：80-120 列
分屏终端：40-60 列
极窄终端：< 40 列

不同宽度需要不同布局策略
```

### 内容适配问题

```
问题1：长消息截断
消息超过终端宽度 → 用户看不到完整内容

问题2：组件重叠
固定宽度组件 → 窄屏时重叠

问题3：信息丢失
关键信息在小屏被隐藏 → 用户困惑

问题4：交互困难
按钮/选项在窄屏不可见 → 无法操作
```

## 长消息截断

### 智能截断

```typescript
function truncateMessage(
  message: string,
  maxWidth: number,
  options?: TruncateOptions
): TruncatedMessage {
  const lines = message.split('\n')
  const truncatedLines: string[] = []
  let totalLines = 0
  const maxLines = options?.maxLines || 10

  for (const line of lines) {
    if (totalLines >= maxLines) {
      // 超过行数限制
      truncatedLines.push('... (更多内容折叠)')
      break
    }

    // 截断每行
    const truncatedLine = truncateLine(line, maxWidth)
    truncatedLines.push(truncatedLine)

    // 多行折算
    const wrappedLines = Math.ceil(line.length / maxWidth)
    totalLines += wrappedLines
  }

  return {
    content: truncatedLines.join('\n'),
    truncated: truncatedLines.length < lines.length ||
               lines.some(l => l.length > maxWidth),
    originalLines: lines.length,
    displayedLines: truncatedLines.length,
  }
}
```

### 行截断

```typescript
function truncateLine(line: string, maxWidth: number): string {
  if (line.length <= maxWidth) {
    return line
  }

  // 截断并添加省略号
  const truncated = line.slice(0, maxWidth - 3)
  return truncated + '...'
}

// 更智能的截断：在空白处截断
function smartTruncateLine(line: string, maxWidth: number): string {
  if (line.length <= maxWidth) {
    return line
  }

  // 寻找最后一个空格
  const lastSpace = line.lastIndexOf(' ', maxWidth - 3)

  if (lastSpace > maxWidth * 0.5) {
    // 在空格处截断
    return line.slice(0, lastSpace) + '...'
  }

  // 直接截断
  return line.slice(0, maxWidth - 3) + '...'
}
```

### 代码截断

```typescript
function truncateCodeBlock(
  code: string,
  maxWidth: number,
  maxLines: number
): TruncatedCode {
  const lines = code.split('\n')

  // 截断行数
  const displayLines = lines.slice(0, maxLines)

  // 截断每行宽度
  const truncatedLines = displayLines.map(line =>
    line.length > maxWidth - 4
      ? line.slice(0, maxWidth - 7) + '...'
      : line
  )

  return {
    content: truncatedLines.join('\n'),
    truncated: lines.length > maxLines,
    totalLines: lines.length,
    displayedLines: truncatedLines.length,
  }
}
```

```typescript
function CollapsibleMessage({ message, config }: CollapsibleProps) {
  const [expanded, setExpanded] = useState(config.defaultExpanded)
  const { columns } = useStdout()
```

### 自动折叠

```typescript
function autoCollapseCheck(
  content: string,
  terminalSize: TerminalSize
): boolean {
  const lines = content.split('\n')
  const threshold = AUTO_COLLAPSE_THRESHOLD[terminalSize]

  // 超过阈值自动折叠
  return lines.length > threshold
}

const AUTO_COLLAPSE_THRESHOLD = {
  full: 50,      // 50 行以上折叠
  standard: 30,  // 30 行以上折叠
  narrow: 15,    // 15 行以上折叠
  minimal: 5,    // 5 行以上折叠
}
```

### 动态状态信息

```typescript
function DynamicStatusBar() {
  const { columns } = useStdout()
  const [priorityInfo, setPriorityInfo] = useState([])

  // 根据宽度确定优先级
  useEffect(() => {
    const priorities = getPriorityInfo(columns)
    setPriorityInfo(priorities)
  }, [columns])

  return (
    <Box>
      {priorityInfo.map((info, i) => (
        <Box key={i}>
          {i > 0 && <Text> | </Text>}
          <Text>{info.label}: {info.value}</Text>
        </Box>
      ))}
    </Box>
  )
}

function getPriorityInfo(columns: number): InfoItem[] {
  // 按优先级排列信息
  const allInfo = [
    { label: 'Claude', value: '', priority: 1 },
    { label: 'Tokens', value: tokenCount, priority: 2 },
    { label: 'Mode', value: mode, priority: 3 },
    { label: 'Session', value: sessionId.slice(0, 8), priority: 4 },
    { label: 'Memory', value: memoryUsage, priority: 5 },
  ]

  // 根据宽度选择
  const maxItems = Math.floor(columns / 20)

  return allInfo
    .slice(0, maxItems)
    .map(info => ({
      label: info.label,
      value: info.value.toString(),
    }))
}
```

## 响应式原则总结

### 1. 尺寸检测

```
实时检测终端宽度 → 选择布局
```

### 2. 智能截断

```
长消息截断，提供展开选项
```

### 3. 自动折叠

```
大内容自动折叠，节省空间
```

### 4. 优先级排序

```
关键信息优先显示，次要信息隐藏
```

### 5. 垂直适配

```
窄屏时改为垂直布局
```
