权限对话框是 Claude Code 与用户交互的关键界面。它既要建立信任，又不能过度打扰。本篇分析这个微妙的设计平衡。

## 权限对话框的双重目标

### 目标 1：建立信任

用户需要知道 AI 想做什么：

```
┌────────────────────────────────────────────────────────────┐
│ ⚠️  Claude 想要执行 Bash 命令                               │
│                                                            │
│ 命令: git push --force origin main                         │
│                                                            │
│ 这将强制推送到主分支，可能覆盖其他人的提交                    │
│                                                            │
│ ┌────────────────────────────────────────────────────────┐ │
│ │ [拒绝] [允许一次] [允许本次会话]                          │ │
│ └────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────┘
```

### 目标 2：减少打扰

不是每个操作都需要确认：

```
┌────────────────────────────────────────────────────────────┐
│ ⚠️  Claude 想要读取文件                                     │
│                                                            │
│ 文件: README.md                                            │
│                                                            │
│ 这只是读取，不修改                                          │
│                                                            │
│ [自动允许此类操作?] ☑                                        │
│                                                            │
│ ┌────────────────────────────────────────────────────────┐ │
│ │ [允许] [允许所有读取]                                     │ │
│ └────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────┘
```

## 对话框信息设计

### 显示内容层次

| 信息层次 | 显示时机 | 重要性 |
|----------|----------|--------|
| 工具类型 | 所有对话框 | 高 |
| 工具参数摘要 | 所有对话框 | 高 |
| 操作影响说明 | 危险操作 | 中 |
| 推理原因 | 可选 | 低 |
| 分类器建议 | 有建议时 | 低 |

### 信息精简原则

终端空间有限，信息要精简：

```
❌ 过多信息：
┌────────────────────────────────────────────────────────────┐
│ ⚠️  Claude 想要执行 Bash 命令                               │
│                                                            │
│ 命令详情:                                                   │
│ git push --force origin main                               │
│                                                            │
│ 工具 ID: tool_abc123                                        │
│ 会话 ID: session_xyz789                                     │
│ 时间戳: 2024-01-15T10:23:45Z                                │
│                                                            │
│ 推理过程:                                                   │
│ Claude 分析了当前分支状态，发现有未推送的提交                │
│ 决定强制推送以同步远程                                      │
│                                                            │
│ AI 分类器建议:                                              │
│ deny (置信度 95%)                                           │
│ 原因: 强制推送到主分支是危险操作                             │
│                                                            │
│ [拒绝] [允许一次] [允许本次会话] [查看完整详情]              │
└────────────────────────────────────────────────────────────┘

✅ 精简信息：
┌────────────────────────────────────────────────────────────┐
│ ⚠️  强制推送到 main 分支                                    │
│                                                            │
│ git push --force origin main                               │
│                                                            │
│ 可能覆盖其他人的提交                                        │
│                                                            │
│ [拒绝] [允许一次] [允许本次会话]                             │
└────────────────────────────────────────────────────────────┘
```

### 工具参数摘要

不同工具有不同的摘要格式：

**Bash**：

```typescript
// 命令可能很长，截断显示
function summarizeBash(command: string): string {
  if (command.length > 50) {
    // 截断，保留关键部分
    return command.slice(0, 50) + '...'
  }
  return command
}
```

**Read**：

```typescript
// 只显示文件名
function summarizeRead(input: { file_path: string }): string {
  return basename(input.file_path)
}
```

**Edit**：

```typescript
// 显示文件 + 修改摘要
function summarizeEdit(input: { file_path: string, old_string: string, new_string: string }): string {
  const file = basename(input.file_path)
  const changeType = input.old_string ? '修改' : '添加'
  return `${file}: ${changeType} ${input.new_string.length} 字符`
}
```

## 用户决策选项

### 选项设计

| 选项 | 效果 | 适用场景 |
|------|------|----------|
| 允许一次 | 只允许这次 | 不常见操作 |
| 允许本次会话 | 同类操作自动允许 | 常见操作（如多次 git） |
| 拒绝 | 阻止执行 | 危险操作 |
| 查看详情 | 查看完整参数 | 需要更多信息 |

### 会话规则记忆

选择'允许本次会话'后添加临时规则：

```typescript
// 用户选择'允许本次会话'
function handleSessionAllow(toolName: string, input: object) {
  // 提取匹配模式
  const pattern = extractPattern(toolName, input)

  // 添加会话规则
  sessionRules.push({
    toolName,
    inputPattern: pattern,
    behavior: 'allow',
    source: 'session_dialog',
  })
}
```

**示例**：

```
用户允许 Bash(git status) 后：
- Bash(git status) → 自动允许
- Bash(git diff) → 自动允许（匹配 git *)
- Bash(npm test) → 需要确认（不匹配）
```

### 规则持久化

用户可以选择永久允许：

```
┌────────────────────────────────────────────────────────────┐
│ ⚠️  Claude 想要读取文件                                     │
│                                                            │
│ 文件: src/auth.ts                                          │
│                                                            │
│ ☑ 记住此决定，以后自动允许读取                              │
│                                                            │
│ [允许并记住] [允许一次] [拒绝]                              │
└────────────────────────────────────────────────────────────┘
```

写入 `settings.json`：

```json
{
  'permissions': {
    'allow': [
      { 'toolName': 'Read', 'behavior': 'allow' }
    ]
  }
}
```

## 推测执行

用户看到对话框时，后台并行运行分类器：

### 并行检查

```typescript
// 显示对话框时启动
function showPermissionDialog(toolName: string, input: object) {
  // 启动推测分类器检查
  startSpeculativeClassifierCheck(toolName, input)

  // 显示对话框
  renderDialog(toolName, input)

  // 监听分类器结果
  watchClassifierResult(toolName, input)
}
```

### 分类器结果处理

```typescript
async function watchClassifierResult(toolName: string, input: object) {
  const result = await classifierPromise

  // 高置信度允许
  if (result.classification === 'allow' && result.confidence > 0.95) {
    // 自动消失对话框
    hideDialog()
    // 自动允许
    executeTool(toolName, input)
    return
  }

  // 高置信度拒绝
  if (result.classification === 'deny' && result.confidence > 0.95) {
    // 更新对话框，显示警告
    updateDialog({
      warning: result.reason,
      suggestion: '拒绝',
    })
    return
  }

  // 不确定，等待用户决定
}
```

**效果**：用户看到对话框，但可能几秒后自动消失（分类器高置信度允许）。

## 对话框动画

### 出现动画

```typescript
// 对话框渐入
function showDialog() {
  // 第一帧：显示标题
  render({ title: '⚠️  Claude 想要执行操作' })

  // 第二帧：显示参数
  await delay(100)
  render({ title, params: summarizeParams() })

  // 第三帧：显示按钮
  await delay(100)
  render({ title, params, buttons: ['允许', '拒绝'] })
}
```

### 消失动画

```typescript
// 对话框消失
function hideDialog() {
  // 渐出效果
  for (let i = 1; i >= 0; i -= 0.1) {
    render({ opacity: i })
    await delay(50)
  }

  // 清除对话框
  clearDialog()
}
```

## 不同工具的对话框

### Read：最小打扰

```
┌────────────────────────────────────────────────────────────┐
│ 📖 Claude 想要读取                                          │
│                                                            │
│ README.md                                                  │
│                                                            │
│ [允许所有读取] [允许一次]                                    │
└────────────────────────────────────────────────────────────┘
```

- 只读操作，风险低
- 选项更简单
- 鼓励批量允许

### Edit：必须确认

```
┌────────────────────────────────────────────────────────────┐
│ ✏️ Claude 想要修改                                          │
│                                                            │
│ src/auth.ts                                                │
│                                                            │
│ 添加 23 字符                                               │
│                                                            │
│ [查看变更] [允许] [拒绝]                                     │
└────────────────────────────────────────────────────────────┘
```

- 修改操作，风险中等
- 提供查看变更选项
- 不提供批量允许

### Bash：详细说明

```
┌────────────────────────────────────────────────────────────┐
│ ⚠️  Claude 惀要执行命令                                     │
│                                                            │
│ git push --force origin main                               │
│                                                            │
│ 影响：可能覆盖 5 个提交                                     │
│                                                            │
│ 分类器建议：拒绝 (高风险)                                   │
│                                                            │
│ [拒绝] [允许一次] [允许本次会话 git push]                    │
└────────────────────────────────────────────────────────────┘
```

- 命令风险高
- 显示影响说明
- 显示分类器建议
- 会话允许限定模式

## 对话框 UX 原则

### 1. 信息充分但不冗余

```
✅ 告诉用户做什么、为什么
❌ 不要显示内部调试信息
```

### 2. 选项清晰

```
✅ 每个选项的效果明确
❌ 不要用模糊的'是/否'
```

### 3. 默认选择安全

```
✅ 默认焦点在'拒绝'或最安全选项
❌ 不要默认聚焦在危险选项
```

### 4. 批量允许要谨慎

```
✅ 读操作可以批量允许
❌ 写操作不建议批量允许
❌ Bash 命令限制批量范围
```

### 5. 分类器辅助

```
✅ 显示分类器建议
✅ 高置信度允许可以自动消失
❌ 不要隐藏分类器的拒绝建议
```

### 6. 会话规则限定范围

```
✅ '允许本次会话 git push' 只匹配 git push
❌ '允许本次会话 Bash' 匹配所有 Bash（太宽泛）
```

### 7. 提供更多信息入口

```
✅ [查看详情] 查看完整参数
❌ 不要默认显示完整详情（太长）
```
