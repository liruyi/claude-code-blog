CLI（命令行界面）一直是开发者的核心工具。但 AI Agent 的出现，彻底改变了 CLI 的交互范式。本系列从用户体验角度，分析 Claude Code 如何设计终端中的 AI 交互。

## 为什么终端 AI 的 UX 特别？

传统 CLI 的交互模式：

```
用户输入命令 → 系统执行 → 输出结果 → 结束
```

这是一个**封闭循环**：
- 用户知道要做什么
- 命令是明确的指令
- 结果是确定的输出

AI CLI 的交互模式：

```
用户描述意图 → Agent 思考 → 可能多次执行 → 可能多个工具 → 可能询问用户 → 最终响应 → 可能继续...
```

这是一个**开放式循环**：
- 用户描述意图，而非具体命令
- Agent 决定如何执行
- 过程可能很长、复杂、不确定

## 传统 CLI vs AI CLI

| 对比 | 传统 CLI | AI CLI |
|------|----------|--------|
| **输入** | 明确命令 | 自然语言描述 |
| **执行** | 单次操作 | 可能多次迭代 |
| **输出** | 确定结果 | 可能不确定 |
| **交互** | 无中途交互 | 可能询问用户 |
| **时长** | 秒级 | 可能分钟级 |
| **用户角色** | 指挥者 | 协作者 |

## AI CLI 的 UX 挑战

### 1. 不确定性带来的焦虑

用户不知道：
- Agent 会做什么操作
- 操作会花多久
- 结果是否正确

**设计对策**：
- 实时显示进度
- 预告即将执行的操作
- 提供中断能力

### 2. 长时间等待的挫败感

传统命令几秒完成，AI 可能几分钟：

```
用户: 重构这个模块
     │
     └─► Agent 思考... 30s
         └─► 读取文件... 10s
             └─► 分析结构... 20s
                 └─► 编辑文件... 15s
                     └─► 运行测试... 30s
                         └─► 修复问题... 20s
                             └─► 完成
```

总时长可能 2 分钟。

**设计对策**：
- 流式显示思考过程
- 分阶段显示进度
- 让用户可以继续聊天（Fork 模式）

### 3. 操作风险的担忧

Agent 可能执行危险操作：

```
用户: 清理临时文件
Agent: 执行 rm -rf ./temp
     │
     └─► 用户担忧：会不会删错？
```

**设计对策**：
- 权限对话框确认
- 显示操作影响范围
- 提供撤销能力（有限）

### 4. 信任建立的困难

用户不信任 AI 的判断：

```
用户: 这个重构正确吗？
Agent: 我认为是的
     │
     └─► 用户怀疑：AI 可能出错
```

**设计对策**：
- 显示推理过程
- 提供验证证据
- 允许用户审查每一步

## Claude Code 的 UX 设计哲学

### 1. 实时反馈 > 最终结果

不是等待完成后显示结果，而是实时显示过程：

```
传统：
[等待 2 分钟]
✅ 重构完成

Claude Code：
📖 正在分析文件结构...
🔧 正在重构 auth.ts...
✅ 已重构 auth.ts
📖 正在分析 api.ts...
🔧 正在重构 api.ts...
✅ 已重构 api.ts
🧪 正在运行测试...
✅ 所有测试通过
```

### 2. 信任建立 > 效率优先

宁可多一步确认，也不要让用户担忧：

```
Agent 想执行：rm -rf ./node_modules

显示对话框：
⚠️  Claude 想要执行：
    rm -rf ./node_modules

    这将删除 500MB 文件

    [Allow] [Deny] [Allow for this session]
```

### 3. 用户控制 > Agent 自主

用户始终可以：

- 中断执行（Ctrl+C）
- 拒绝操作
- 修改 Agent 的决定
- 查看执行历史

### 4. 简洁信息 > 详细日志

终端空间有限，信息要精简：

```
❌ 不好的设计：
[2024-01-15 10:23:45] INFO: Starting file analysis
[2024-01-15 10:23:46] INFO: File count: 15
[2024-01-15 10:23:47] DEBUG: Scanning src/auth.ts
[2024-01-15 10:23:48] DEBUG: Scanning src/api.ts
...

✅ 好的设计：
📖 分析 15 个文件
```

## 设计原则预告

本系列将提炼 10 条 AI CLI UX 设计原则：

1. **实时反馈原则**：永远不要让用户等待沉默
2. **信任建立原则**：每个操作都要解释原因
3. **用户控制原则**：用户随时可以干预
4. **信息精简原则**：终端空间有限，只显示关键信息
5. **进度可见原则**：长任务必须有进度指示
6. **中断友好原则**：Ctrl+C 不应该丢失工作
7. **错误友好原则**：错误消息要可理解、可解决
8. **引导渐进原则**：新用户不需要读文档
9. **恢复无缝原则**：会话恢复感觉像继续
10. **响应式原则**：适配不同终端大小
