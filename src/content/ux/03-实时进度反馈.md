长任务是 AI CLI 的常态。如果用户只看到空白等待，会产生焦虑。本篇分析 Claude Code 如何设计实时进度反馈，让等待变得可预期。

## 为什么实时反馈重要？

### 用户心理

等待时的心理状态：

```
空白等待：
用户看到什么都没有 → 焦虑 → 不信任 → 想中断 → 体验差

实时反馈：
用户看到进度 → 了解状态 → 有预期 → 安心等待 → 体验好
```

### 信息需求层次

用户在等待时的信息需求：

| 需求 | 满足方式 |
|------|----------|
| **知道在做** | Spinner + 文字 '正在处理...' |
| **知道做什么** | 工具名称 + 参数摘要 |
| **知道进度** | 百分比 + 已完成数量 |
| **知道何时完成** | 剩余时间估算（最难） |
| **知道能否继续其他** | Fork 模式提示 |

### Spinner 动画

状态栏使用 Spinner 表示活动状态：

```
思考时：○ (空心圆，旋转)
输出时：● (实心圆)
工具执行：⟳ (旋转箭头)
等待输入：▸ (箭头)
```

**实现**：使用 Ink 的 `<Spinner>` 组件：

```typescript

```

### 工具进度显示

每个工具显示自己的进度：

**短工具**（< 1s）：不显示进度，只显示状态

```
✓ Read: src/auth.ts (0.5s)
```

**长工具**（> 1s）：显示进度百分比

```
🔄 Bash: npm test ──────────────────────── 45%
```

**进度来源**：

| 工具 | 进度来源 |
|------|----------|
| Bash | 解析命令输出中的进度指示 |
| Read | 文件大小 + 已读字节 |
| WebFetch | HTTP 响应进度（不稳定） |
| Agent | 子 Agent 轮次 |

## Bash 进度解析

Bash 命令的进度最复杂，需要解析输出：

### 测试命令进度

```typescript
// npm test 输出解析
function parseTestProgress(output: string): number {
  // 模式匹配
  const match = output.match(/(\d+) passing/)
  if (match) {
    const passed = parseInt(match[1])
    // 需要知道总数（从之前的统计）
    return (passed / totalTests) * 100
  }

  // Jest 输出
  const jestMatch = output.match(/(\d+)\/(\d+) test files/)
  if (jestMatch) {
    return (parseInt(jestMatch[1]) / parseInt(jestMatch[2])) * 100
  }

  return 0  // 无法解析
}
```

### 编译命令进度

```typescript
// TypeScript 编译进度
function parseTscProgress(output: string): number {
  const match = output.match(/(\d+)\/(\d+) files/)
  if (match) {
    return (parseInt(match[1]) / parseInt(match[2])) * 100
  }

  // 最后检查
  if (output.includes('Done in')) {
    return 100
  }

  return 0
}
```

### Git 命令进度

```typescript
// git clone/fetch 进度（自带）
function parseGitProgress(output: string): number {
  // Git 输出自带进度
  // Receiving objects: 45% (1234/5678)
  const match = output.match(/Receiving objects: (\d+)%/)
  if (match) return parseInt(match[1])

  const match2 = output.match(/(\d+)\/(\d+)/)
  if (match2) {
    return (parseInt(match2[1]) / parseInt(match2[2])) * 100
  }

  return 0
}
```

## Streaming Tool Executor

Claude Code 的创新：**在 API 流式返回时就开始执行工具**

### 传统流程

```
API 返回全部内容 → 解析所有工具调用 → 依次执行工具 → 返回结果
```

问题：API 返回可能需要 10s，工具才开始执行。

### Streaming 流程

```
API 开始流式返回 → 解析到第一个工具 → 开始执行 → 继续解析 → 执行更多 → 最终完成
```

**效果**：工具执行提前开始，减少等待时间。

### 实现

```typescript
// Streaming Tool Executor
class StreamingToolExecutor {
  private pendingTools: Map<string, Promise<ToolResult>>
  private completedResults: Map<string, ToolResult>

  // 添加工具（在 API 流式返回时）
  addTool(toolBlock: ToolUseBlock, partialMessage: Message) {
    const toolPromise = this.executeTool(toolBlock)
    this.pendingTools.set(toolBlock.id, toolPromise)
  }

  // 获取已完成结果（流式返回）
  getCompletedResults(): ToolResult[] {
    const completed = []
    for (const [id, promise] of this.pendingTools) {
      if (promise.status === 'fulfilled') {
        completed.push(promise.value)
        this.pendingTools.delete(id)
      }
    }
    return completed
  }

  // 执行单个工具
  private async executeTool(block: ToolUseBlock): Promise<ToolResult> {
    const tool = getTool(block.name)
    const result = await tool.call(block.input, context)
    return result
  }
}
```

**关键**：工具执行与 API 流同时进行。

### 使用

```typescript
async function* query(messages, context) {
  const streamingExecutor = new StreamingToolExecutor()

  for await (const event of streamEvents) {
    if (event.type === 'content_block_start' && event.content_block.type === 'tool_use') {
      // 发现工具调用，立即添加
      streamingExecutor.addTool(event.content_block, message)
    }

    // 同时获取已完成结果
    for (const result of streamingExecutor.getCompletedResults()) {
      yield { type: 'tool_result', result }
    }

    // 文本块直接 yield
    if (event.type === 'content_block_delta') {
      yield { type: 'text', content: event.delta.text }
    }
  }

  // 获取剩余结果
  for (const result of streamingExecutor.getRemainingResults()) {
    yield { type: 'tool_result', result }
  }
}
```

## 进度更新频率

进度更新不能太频繁（终端刷新率限制）：

```typescript
// 进度更新间隔
const PROGRESS_UPDATE_INTERVAL = 500  // 500ms

// 节流更新
let lastProgressUpdate = 0

function updateProgress(progress: number) {
  const now = Date.now()
  if (now - lastProgressUpdate < PROGRESS_UPDATE_INTERVAL) {
    return  // 太频繁，跳过
  }
  lastProgressUpdate = now

  // 实际更新 UI
  renderProgress(progress)
}
```

## AsyncGenerator 模式

Claude Code 使用 AsyncGenerator 实现流式：

```typescript
async function* query(options: QueryOptions): AsyncGenerator<Message> {
  while (true) {
    // 准备请求
    const messages = prepareMessages(options)

    // 流式调用 API
    for await (const event of callModel(messages)) {
      // 处理事件，yield 消息
      if (event.type === 'text') {
        yield { type: 'assistant', content: event.text }
      }

      if (event.type === 'tool_use') {
        // 执行工具
        const result = await executeTool(event.tool)
        yield { type: 'tool_result', result }
      }
    }

    // 决定继续或退出
    if (shouldContinue()) {
      // 更新状态，继续循环
      options.messages = updatedMessages
    } else {
      return  // 结束
    }
  }
}
```

**优势**：
- 流式 yield，不阻塞
- 支持 Ctrl+C 中断
- 支持 yield 间继续聊天

## 中断处理

用户 Ctrl+C 时的处理：

```typescript
// 设置 AbortController
const abortController = new AbortController()

// 监听中断
process.on('SIGINT', () => {
  abortController.abort()
})

// 传递给工具
async function executeTool(tool, input, abortSignal) {
  if (abortSignal.aborted) {
    return { type: 'cancelled' }
  }

  // 工具内部也要监听中断
  const result = await tool.call(input, { signal: abortSignal })

  return result
}
```

### 中断后的恢复

中断不丢失工作：

```typescript
// 中断时保存状态
function handleInterruption() {
  // 保存当前进度
  saveProgress(currentState)

  // 保存已完成的工作
  saveCompletedWork(completedResults)

  // 提示用户
  console.log('已暂停。使用 /resume 继续')
}
```

## 进度设计原则

### 1. 永远不让用户看到空白

```
❌ 空白等待
[等待 30s]
✅ 结果

✅ 实时反馈
🔄 正在分析...
🔄 读取文件 23/45...
✅ 结果
```

### 2. 告诉用户做什么

```
❌ 模糊
正在处理...

✅ 具体
正在分析项目结构
正在读取 src/auth.ts
```

### 3. 告诉用户进度

```
❌ 无进度
正在执行测试...

✅ 有进度
正在执行测试 45/100 (45%)
```

### 4. 不承诺精确时间

```
❌ 不精确估算
预计剩余 30s（可能不准）

✅ 已知信息
已执行 45/100 测试
```

### 5. 流式优于批量

```
❌ 等待完成
等待所有工具执行完毕 → 显示所有结果

✅ 流式显示
执行第一个 → 显示结果 → 执行第二个 → 显示结果
```
