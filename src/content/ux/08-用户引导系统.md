新用户需要引导，老用户需要发现新功能。本篇分析 Claude Code 的用户引导设计。

## 引导的挑战

### 用户类型差异

```
新用户：
- 不知道如何开始
- 不了解工具能力
- 担心安全问题

中级用户：
- 知道基本功能
- 不了解高级特性
- 可能错过效率技巧

高级用户：
- 熟悉大部分功能
- 可能错过新功能
- 需要定制化引导
```

### CLI 引导限制

```
CLI vs Web/App：
- 无动画引导
- 无 tooltip 提示
- 无鼠标交互
- 受终端宽度限制
- 文本为主
```

function ContextualHint({ context }: HintContextProps) {
  // 根据上下文选择提示
  const hint = selectHint(context)

  if (!hint) return null

  // 检查是否已显示
  if (hasShownHint(hint.id)) return null

  return (
    <Box marginTop={1}>
      <FoldedHint hint={hint} />
    </Box>
  )
}

function selectHint(context: HintContext): Hint | null {
  const hints = [
    {
      id: 'first-edit',
      trigger: { tool: 'Edit', count: 1 },

      content: 'AI 会先读取文件，然后进行编辑。可以随时撤销。',

      title: '执行命令',
      content: '危险命令会请求确认。可以设置规则自动允许。',

      trigger: { mcpConnection: true },

}

### 提示频率控制

```typescript
const HINT_CONFIGS = {
  maxHintsPerSession: 3,    // 每次会话最多 3 个提示
  hintCooldown: 300000,     // 提示间隔 5 分钟
  neverRepeat: true,        // 同一提示不重复
}

function shouldShowHint(hintId: string, context: HintContext): boolean {
  // 检查本会话提示数量
  if (context.sessionHintsShown >= HINT_CONFIGS.maxHintsPerSession) {
    return false
  }

  // 检查冷却时间
  const lastHintTime = getLastHintTime()
  if (Date.now() - lastHintTime < HINT_CONFIGS.hintCooldown) {
    return false
  }

  // 检查是否已显示
  if (HINT_CONFIGS.neverRepeat && hasShownHint(hintId)) {
    return false
  }

  return true
}
```

## 进阶引导

### 功能发现提示

```typescript
function FeatureDiscovery() {
  // 检测用户是否使用了某功能
  const usedFeatures = getUsedFeatures()

  // 推荐未使用的相关功能
  const recommendations = recommendFeatures(usedFeatures)

  if (recommendations.length === 0) return null

  return (
    <Box flexDirection='column'>
      <Text bold>💡 发现新功能</Text>
      {recommendations.slice(0, 2).map(rec => (
        <Box key={rec.id} flexDirection='column'>
          <Text>{rec.title}</Text>
          <Text dimColor>{rec.description}</Text>
        </Box>
      ))}
    </Box>
  )
}

function recommendFeatures(used: string[]): Feature[] {
  const allFeatures = [
    {
      id: 'skills',
      title: 'Skills（技能）',
      description: '预定义任务，如 /commit、/review-pr',
      related: ['edit', 'bash', 'git'],
    },
    {
      id: 'mcp',
      title: 'MCP Server',
      description: '外部工具集成，如数据库、API',
      related: ['read', 'search'],
    },
    {
      id: 'hooks',
      title: 'Hooks（钩子）',
      description: '自定义事件响应',
      related: ['config'],
    },
  ]

  // 推荐相关但未使用的功能
  return allFeatures.filter(f =>
    !used.includes(f.id) &&
    f.related.some(r => used.includes(r))
  )
}
```

## 引导原则总结

### 1. 首次体验流畅

```
快速引导，不强制完成所有步骤
```

### 2. 上下文相关

```
根据用户操作显示相关提示
```

### 3. 频率控制

```
不频繁打扰，间隔合理
```

### 4. 可折叠

```
提示可折叠，不占用过多空间
```

### 5. 功能发现

```
推荐相关但未使用的功能
```

### 6. 帮助随时可用

```
/help 命令提供完整帮助
```
